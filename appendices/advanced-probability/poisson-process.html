The Poisson Process

  • Show All Code
  • Hide All Code
  • View Source

The mathematics of things that happen at random in time — arrivals, decays, rare events — and why the waiting between them is always exponential.

Customers entering a bank, radioactive atoms decaying, earthquakes striking: events that pop up at random in continuous time. The Poisson process is the canonical model — a process that, at every instant, “probabilistically restarts itself.” Count its events in any window and you get a Poisson distribution; measure the gaps between events and you get the exponential — the unique memoryless waiting time. One process ties those two distributions together.

Counting processes

A counting process \{N(t),\,t\ge 0\} records the total number of events that have occurred up to time t. (Ross 2014) The Poisson process of rate \lambda>0 is the most important one, pinned down by two intuitive properties: (Ross 2014)

  • Independent increments — counts in disjoint intervals are independent. What happens between 2:00 and 3:00 tells you nothing about 5:00–6:00.
  • Stationary increments — the distribution of the count in an interval depends only on its length, not when it sits. A one-hour window is statistically the same at noon or midnight.

From these, the number of events in an interval of length t is Poisson with mean \lambda t: (Ross 2014) \Pr\{N(t)=n\} = e^{-\lambda t}\,\frac{(\lambda t)^n}{n!}, \qquad n=0,1,2,\dots The rate \lambda is events per unit time; over a window of length t you expect \lambda t of them.

Interarrival times are exponential — and that’s no accident

Look at the gaps between successive events — the interarrival times T_1, T_2, \dots. Ross’s result: they are independent and identically distributed Exponential(\lambda), each with mean 1/\lambda. (Ross 2014) This is the process’s deepest fact, and it follows from a single idea: stationary + independent increments mean that “at any point in time, the process probabilistically restarts itself.” (Ross 2014) Since the exponential is the unique memoryless continuous distribution, it is the only law a restart-anywhere process can have — the time to the next event cannot depend on how long you’ve already waited. (Ross 2014) (This is exactly the constant-hazard, exponential-survival baseline, seen from the other side.)

Show the figure code 
import numpy as np
import matplotlib.pyplot as plt
from math import factorial

rng = np.random.default_rng(2)
lam = 3.0
T = 20.0
# Generate arrivals by summing Exponential interarrival times
gaps = rng.exponential(1/lam, size=int(lam*T*1.5))
times = np.cumsum(gaps)
times = times[times < T]

fig, (axT, axB) = plt.subplots(2, 1, figsize=(7.6, 5.0))

axT.step(np.concatenate([[0], times]), np.arange(len(times)+1), where="post", color="#1f3a5f", lw=1.6)
axT.plot(times, np.zeros_like(times), "|", color="#c0392b", markersize=12, markeredgewidth=1.5)
axT.set_xlim(0, T); axT.set_xlabel("time  t"); axT.set_ylabel("N(t)")
axT.set_title(f"Counting process N(t), rate λ={lam}  (red ticks = events)")

# Counts in fixed windows of width w
w = 1.0
edges = np.arange(0, T+w, w)
counts = np.histogram(times, bins=edges)[0]
ks = np.arange(0, counts.max()+2)
pmf = np.exp(-lam*w) * (lam*w)**ks / np.array([factorial(k) for k in ks])
axB.hist(counts, bins=np.arange(-0.5, counts.max()+1.5), density=True, color="#9bb4d4",
         alpha=0.85, label="observed per-window counts")
axB.plot(ks, pmf, "o-", color="#c0392b", lw=1.8, label=f"Poisson(λw={lam*w:g}) pmf")
axB.set_xlabel("events per window"); axB.set_ylabel("proportion")
axB.set_title("Counts per fixed window are Poisson")
axB.legend(fontsize=8, frameon=False)
fig.tight_layout()
plt.show()

Superposition and thinning

Two operations make the Poisson process beautifully closed: (Ross 2014)

  • Superposition. Merge two independent Poisson processes with rates \lambda_1 and \lambda_2 and the combined stream is Poisson with rate \lambda_1+\lambda_2. (All the arrivals from two doors into one shop.)
  • Thinning. Independently tag each event “Type I” with probability p or “Type II” with 1-p; the two sub-streams are independent Poisson processes with rates \lambda p and \lambda(1-p). (Ross 2014) (Split arrivals into “web” vs “phone” and each is still Poisson.)

These closure rules are why the Poisson process is the default model whenever independent streams combine or split.

Where it shows up

Ross names the Poisson process the standard model for “rare events” and arrivals: radioactive \alpha-particle emissions, earthquakes, the outbreak of wars, and customers entering a bank or post office. (Ross 2014) In this book it is the continuous-time engine beneath count-data models (a Poisson regression models the rate \lambda) and survival analysis (the exponential interarrival time is the constant-hazard duration).

Common pitfalls

WarningWhere this goes wrong

  • Assuming Poisson when events clump. Independent increments fail if one event makes the next more likely (“contagion”) — that’s overdispersion; use a different process / negative binomial. (Ross 2014)

  • Forgetting the memoryless catch. “It’s been quiet, so an arrival is overdue” is the gambler’s fallacy in continuous time — the exponential has no memory. (Ross 2014)

  • Mismatching rate and window. The count is Poisson(\lambda t), not Poisson(\lambda); keep the time units straight (the offset issue). (Ross 2014)
