Boosting: AdaBoost, Gradient Boosting & XGBoost

  • Show All Code
  • Hide All Code
  • View Source

Bagging averages many independent trees to kill variance. Boosting does the opposite — grows them one at a time, each fixing the last one’s mistakes — and is the strongest off-the-shelf predictor on tabular data.

Random forests build many trees in parallel and average them. Boosting turns the idea inside out: grow trees sequentially, each new (weak) tree fitting the part the current ensemble still gets wrong, and add it in scaled by a small learning rate. Learning slowly from residuals — via gradient descent in the space of functions — is what makes gradient boosting and its turbocharged cousin XGBoost the methods that win tabular-data competitions.

Why it matters

On structured/tabular social-science data, gradient-boosted trees are routinely the most accurate off-the-shelf predictor — ahead of deep nets and forests. Understanding why (and how to tune the three knobs without overfitting) is essential ML literacy, and it cements the bias–variance lesson: forests cut variance, boosting cuts bias by learning slowly.

Sequential, not parallel

The contrast with bagging is the whole idea (James et al. 2021):

  • Bagging fits many independent, high-variance trees on bootstrap samples and averages them — parallel variance reduction.
  • Boosting fits trees sequentially, each “grown using information from previously grown trees” — no bootstrap; each tree is fit to a modified version of the data (the current errors) (James et al. 2021).

By adding many weak learners (high-bias, low-variance — often shallow trees) scaled by a small learning rate, the ensemble “learns slowly,” which typically yields superior predictive performance (James et al. 2021).

AdaBoost: focus on the hard cases

AdaBoost (Adaptive Boosting) was the first big success (James et al. 2021). For classification, it weights every observation; after each round it increases the weights of the misclassified points, forcing the next weak learner to concentrate on the hard cases (James et al. 2021). The final prediction is a weighted majority vote, with more accurate learners given more influence (\alpha_m) (James et al. 2021). Statistically, AdaBoost minimises an exponential loss — which makes it sensitive to outliers/noise (it penalises misclassification exponentially), a weakness modern methods improve on (James et al. 2021).

Gradient boosting: gradient descent in function space

Gradient boosting generalises the idea as numerical optimisation: minimise a loss by adding a function at each step (James et al. 2021). Each new tree is fit to the negative gradient of the loss at the current predictions — and for squared-error loss the negative gradient is simply the residuals (James et al. 2021). The algorithm (ISL Alg. 8.2) (James et al. 2021; Jacobucci et al. 2023):

Initialise \hat f(x)=0 and residuals r_i=y_i. For b = 1,\dots,B: (a) fit a shallow tree \hat f^b (with d splits) to the residuals (X,r); (b) update \hat f(x) \leftarrow \hat f(x) + \lambda\,\hat f^b(x); (c) update residuals r_i \leftarrow r_i - \lambda\,\hat f^b(x_i). Output \hat f(x) = \sum_{b=1}^{B}\lambda\,\hat f^b(x).

Show the code (gradient boosting with stumps, by hand) 
import numpy as np
import matplotlib.pyplot as plt

rng = np.random.default_rng(0)
f_true = lambda x: np.sin(2*x) + 0.5*x
n = 200
x = rng.uniform(0, 6, n); y = f_true(x) + rng.normal(0, 0.5, n)
xt = rng.uniform(0, 6, n); yt = f_true(xt) + rng.normal(0, 0.5, n)
grid = np.linspace(0, 6, 300)

def fit_stump(x, r):
    o = np.argsort(x); xs, rs = x[o], r[o]
    csum = np.cumsum(rs); tot = csum[-1]; m = len(rs)
    k = np.arange(1, m)
    lsum, rsum = csum[:-1], tot - csum[:-1]
    gain = lsum**2 / k + rsum**2 / (m - k)         # RSS reduction
    b = np.argmax(gain)
    return (xs[b] + xs[b+1]) / 2, lsum[b]/k[b], rsum[b]/(m-k[b])

def pred(stump, x):
    thr, cL, cR = stump
    return np.where(x <= thr, cL, cR)

# Left panel: staged fits at lambda = 0.1
lam = 0.1; B = 200; f = np.zeros(n); snaps = {1, 10, 50, 200}; curves = {}
fg = np.zeros(len(grid))
for b in range(1, B+1):
    s = fit_stump(x, y - f); f += lam * pred(s, x); fg += lam * pred(s, grid)
    if b in snaps: curves[b] = fg.copy()

fig, (axL, axR) = plt.subplots(1, 2, figsize=(9.2, 4.0))
axL.scatter(x, y, s=8, color="#cdd6df")
axL.plot(grid, f_true(grid), color="#1f8a4c", lw=2.4, label="true f(x)")
for b, c in sorted(curves.items()):
    axL.plot(grid, c, lw=1.4, alpha=0.9, label=f"B = {b}")
axL.set_title("Staged fit: learning slowly from residuals"); axL.set_xlabel("x"); axL.set_ylabel("y")
axL.legend(fontsize=7.5, frameon=False)

# Right panel: train/test error vs B for two learning rates
axR.axhline(0, color="none")
for lam, col in [(0.1, "#1f4e79"), (0.5, "#c0392b")]:
    f = np.zeros(n); fte = np.zeros(n); tr, te = [], []
    for b in range(300):
        s = fit_stump(x, y - f); f += lam*pred(s, x); fte += lam*pred(s, xt)
        tr.append(np.mean((y-f)**2)); te.append(np.mean((yt-fte)**2))
    axR.plot(tr, color=col, lw=1.5, ls="--", alpha=0.7)
    axR.plot(te, color=col, lw=2.0, label=f"test, λ={lam}")
axR.set_title("Boosting can overfit (test error, solid)"); axR.set_xlabel("number of trees B")
axR.set_ylabel("MSE"); axR.legend(fontsize=8, frameon=False); axR.set_ylim(0, 1.2)
fig.tight_layout()
plt.show()

The three knobs

Gradient boosting is tuned by three hyperparameters (James et al. 2021):

  • Number of trees B. Unlike bagging, boosting can overfit if B is too large (though usually slowly) — so B must be chosen, typically by cross-validation (James et al. 2021).
  • Learning rate / shrinkage \lambda. A small positive number (often 0.01–0.1). Smaller \lambda needs a larger B but generalises better (James et al. 2021).
  • Interaction depth d. The number of splits per tree; d=1 (stumps) gives an additive model (each term one variable), and larger d allows higher-order interactions (James et al. 2021).

The recurring recipe: small \lambda, many trees, modest depth, CV-chosen B.

XGBoost: the competition workhorse

XGBoost (Extreme Gradient Boosting) is the efficient, regularised implementation that dominates tabular-data competitions (James et al. 2021; Bruce et al. 2020):

  • Regularisation — its objective penalises tree complexity, curbing overfitting (Bruce et al. 2020).
  • Second-order (Newton) updates — uses both the gradient and the Hessian of the loss, not just the first derivative (Bruce et al. 2020).
  • Stochastic features — row and column subsampling decorrelate trees and speed training (Bruce et al. 2020).
  • Early stopping — halt once validation performance stops improving (Bruce et al. 2020).

Reading a boosted model

Because the ensemble is hundreds of trees, you can’t draw it — so you interpret it indirectly (James et al. 2021):

  • Variable importance — total reduction in RSS (or Gini) from splits on each predictor, averaged over all B trees (James et al. 2021).
  • Partial-dependence plots — the marginal effect of one or two predictors after integrating out the rest (James et al. 2021) (see interpretability & ML).

Common pitfalls

WarningWhere this goes wrong
  • Letting B run unbounded. Boosting overfits with too many trees — cross-validate B / use early stopping (bagging never has this problem). (James et al. 2021)
  • Too-large a learning rate. Aggressive \lambda fits noise fast; prefer small \lambda with more trees. (James et al. 2021)
  • Deep trees by default. Large d models high-order interactions you may not have data for; start with stumps/shallow trees. (James et al. 2021)
  • AdaBoost on noisy data. Exponential loss chases outliers; gradient boosting with a robust loss (or XGBoost regularisation) is safer. (James et al. 2021)
  • Trusting a single importance number. Importances are split-based and correlated-feature-sensitive; corroborate with partial-dependence/SHAP. (James et al. 2021)

Appendix · the mathematics

Functional gradient descent. To minimise \sum_i L(y_i, F(x_i)) over functions F, take a step in the steepest-descent direction in function space: the negative gradient at observation i is g_i = -\big[\partial L(y_i,F(x_i))/\partial F(x_i)\big]. Boosting fits a tree \hat f^b to the g_i and updates F \leftarrow F + \lambda \hat f^b. For squared loss L=\tfrac12(y-F)^2, g_i = y_i - F(x_i) — the residual — recovering the residual-fitting algorithm in the figure.

AdaBoost = forward stagewise on exponential loss. Minimising \sum_i \exp(-y_i F(x_i)) (y_i\in\{-1,1\}) one term at a time yields exactly AdaBoost’s reweighting w_i \leftarrow w_i\exp(\alpha_m\mathbf 1\{y_i\ne G_m(x_i)\}) and learner weight \alpha_m = \tfrac12\log\frac{1-\text{err}_m}{\text{err}_m} — the statistical view that demystified the algorithm.

XGBoost’s second-order objective. At stage t XGBoost approximates the loss by a 2nd-order Taylor expansion, \sum_i [g_i f_t(x_i) + \tfrac12 h_i f_t(x_i)^2] + \Omega(f_t), with gradient g_i, Hessian h_i, and complexity penalty \Omega(f)=\gamma T + \tfrac12\lambda\|w\|^2 (number of leaves T, leaf weights w). The closed-form optimal leaf weight w_j^* = -\frac{\sum_{i\in j} g_i}{\sum_{i\in j} h_i+\lambda} is what makes it fast and regularised.

## The three knobs

Gradient boosting is tuned by three hyperparameters [@islr]:

- **Number of trees $B$.** *Unlike bagging, boosting can **overfit** if $B$ is too large* (though usually
  slowly) — so $B$ must be chosen, typically by cross-validation [@islr].
- **Learning rate / shrinkage $\lambda$.** A small positive number (often **0.01–0.1**). Smaller $\lambda$
  needs a **larger $B$** but **generalises better** [@islr].
- **Interaction depth $d$.** The number of splits per tree; **$d=1$ (stumps)** gives an **additive model**
  (each term one variable), and larger $d$ allows higher-order **interactions** [@islr].

The recurring recipe: **small $\lambda$, many trees, modest depth, CV-chosen $B$.**

## XGBoost: the competition workhorse

**XGBoost** (Extreme Gradient Boosting) is the efficient, regularised implementation that **dominates
tabular-data competitions** [@islr; @bruce_practical]:

- **Regularisation** — its objective **penalises tree complexity**, curbing overfitting [@bruce_practical].
- **Second-order (Newton) updates** — uses both the **gradient and the Hessian** of the loss, not just the
  first derivative [@bruce_practical].
- **Stochastic features** — **row and column subsampling** decorrelate trees and speed training
  [@bruce_practical].
- **Early stopping** — halt once **validation** performance stops improving [@bruce_practical].

## Reading a boosted model

Because the ensemble is hundreds of trees, you can't draw it — so you interpret it indirectly [@islr]:

- **Variable importance** — total reduction in **RSS (or Gini)** from splits on each predictor, averaged over
  all $B$ trees [@islr].
- **Partial-dependence plots** — the marginal effect of one or two predictors after integrating out the rest
  [@islr] (see [interpretability & ML](../04-predictive/interpretability-and-ml.qmd)).

## Common pitfalls

::: {.callout-warning title="Where this goes wrong"}
- **Letting $B$ run unbounded.** Boosting overfits with too many trees — cross-validate $B$ / use early
  stopping (bagging never has this problem). [@islr]
- **Too-large a learning rate.** Aggressive $\lambda$ fits noise fast; prefer small $\lambda$ with more
  trees. [@islr]
- **Deep trees by default.** Large $d$ models high-order interactions you may not have data for; start with
  stumps/shallow trees. [@islr]
- **AdaBoost on noisy data.** Exponential loss chases outliers; gradient boosting with a robust loss (or
  XGBoost regularisation) is safer. [@islr]
- **Trusting a single importance number.** Importances are split-based and correlated-feature-sensitive;
  corroborate with partial-dependence/SHAP. [@islr]
:::

## Appendix · the mathematics {.unnumbered}

::: {.math-appendix}

**Functional gradient descent.** To minimise $\sum_i L(y_i, F(x_i))$ over functions $F$, take a step in the
steepest-descent direction in *function space*: the negative gradient at observation $i$ is
$g_i = -\big[\partial L(y_i,F(x_i))/\partial F(x_i)\big]$. Boosting fits a tree $\hat f^b$ to the $g_i$ and
updates $F \leftarrow F + \lambda \hat f^b$. For squared loss $L=\tfrac12(y-F)^2$, $g_i = y_i - F(x_i)$ — the
**residual** — recovering the residual-fitting algorithm in the figure.

**AdaBoost = forward stagewise on exponential loss.** Minimising $\sum_i \exp(-y_i F(x_i))$ ($y_i\in\{-1,1\}$)
one term at a time yields exactly AdaBoost's reweighting $w_i \leftarrow w_i\exp(\alpha_m\mathbf 1\{y_i\ne
G_m(x_i)\})$ and learner weight $\alpha_m = \tfrac12\log\frac{1-\text{err}_m}{\text{err}_m}$ — the statistical
view that demystified the algorithm.

**XGBoost's second-order objective.** At stage $t$ XGBoost approximates the loss by a 2nd-order Taylor
expansion, $\sum_i [g_i f_t(x_i) + \tfrac12 h_i f_t(x_i)^2] + \Omega(f_t)$, with gradient $g_i$, Hessian
$h_i$, and complexity penalty $\Omega(f)=\gamma T + \tfrac12\lambda\|w\|^2$ (number of leaves $T$, leaf
weights $w$). The closed-form optimal leaf weight $w_j^* = -\frac{\sum_{i\in j} g_i}{\sum_{i\in j} h_i+\lambda}$
is what makes it fast and regularised.

:::
