Deep Learning: Neural Networks

  • Show All Code
  • Hide All Code
  • View Source

The canonical black box, demystified: a network is just a stack of simple nonlinear units that learn their own features — powerful on images and text, oversold on a spreadsheet.

Part IV kept naming neural networks as the far, opaque end of the flexibility–interpretability frontier. Here is what’s inside the box. A neuron is a weighted sum passed through a nonlinear switch; stack thousands across layers and the network learns its own features — edges, then shapes, then faces — instead of you engineering them. That representation-learning is why deep nets conquered vision and language. It is also why, on a small social-science table, they usually have nothing to offer a random forest doesn’t.

Why it matters

Deep learning is the most consequential ML development of the century, and the one most surrounded by hype. (James et al. 2021) A social scientist needs to understand it well enough to (a) use it when the data really is unstructured (text, images, audio) and (b) resist it when a linear model or forest would do as well and explain itself. (Jacobucci et al. 2023) This entry demystifies the architecture and the training — and draws the honest line about when it actually wins.

The neuron

The atom of a network is the neuron: take inputs Z, form a weighted sum plus a bias, and pass it through a nonlinear activation function \sigma: (James et al. 2021) X_m=\sigma\big(\alpha_{m0}+\alpha_m^\top Z\big). That nonlinearity is non-negotiable: without it, stacking layers just composes linear maps into one linear map, and the whole network collapses to a regression. (James et al. 2021) The two activations to know: (James et al. 2021)

  • Sigmoid \sigma(v)=1/(1+e^{-v}) — the logistic squash to (0,1), classic but prone to vanishing gradients.
  • ReLU \sigma(v)=\max(0,v) — the modern default: cheap, and effective in deep networks. Each ReLU is a hinge, and a sum of hinges builds a piecewise-linear curve (the figure).

Feed-forward networks: features, learned

A feed-forward network stacks hidden layers between input and output. (James et al. 2021) The key idea: each hidden layer’s units are learned features — “constructed regressors,” representations, embeddings — built automatically from the raw inputs rather than hand-crafted. (James et al. 2021) The output layer reads off the final representation: linear for regression, softmax for classification (probabilities over mutually exclusive classes). (James et al. 2021)

Two facts frame the power: (James et al. 2021)

  • Universal approximation. A single hidden layer with enough units can approximate any reasonable function arbitrarily well (Kuan & White, 1994). Flexibility is not the question — control is.
  • Depth buys hierarchy. Width adds raw flexibility, but depth lets the network compose simple patterns into complex ones — early layers detect edges, later layers assemble them into objects. Hierarchical features are what make deep learning more than wide learning. (James et al. 2021)
Show the figure code 
import numpy as np
import matplotlib.pyplot as plt

fig, (axL, axR) = plt.subplots(1, 2, figsize=(7.8, 3.9))

# ---- Left: network schematic ----
layers = [3, 5, 2]
xpos = [0.1, 0.5, 0.9]
for li, (n, x) in enumerate(zip(layers, xpos)):
    ys = np.linspace(0.15, 0.85, n)
    for y in ys:
        c = {0: "#1f3a5f", 1: "#d98a29", 2: "#1f8a4c"}[li]
        axL.add_patch(plt.Circle((x, y), 0.045, color=c, zorder=3))
    if li > 0:
        prev_ys = np.linspace(0.15, 0.85, layers[li-1])
        for y in ys:
            for py in prev_ys:
                axL.plot([xpos[li-1], x], [py, y], color="#cfd3d8", lw=0.6, zorder=1)
axL.text(0.1, 0.95, "inputs", ha="center", fontsize=8, color="#1f3a5f")
axL.text(0.5, 0.95, "hidden\n(learned features)", ha="center", fontsize=8, color="#b9791f")
axL.text(0.9, 0.95, "output", ha="center", fontsize=8, color="#1f7a44")
axL.set_xlim(0, 1); axL.set_ylim(0.05, 1.05); axL.axis("off")
axL.set_title("Feed-forward network")

# ---- Right: a tiny ReLU net trained on 1-D data ----
rng = np.random.default_rng(0)
x = np.linspace(-3, 3, 120); ytrue = np.sin(1.2 * x) + 0.3 * x
y = ytrue + rng.normal(0, 0.15, x.size)
Xin = x[:, None]
H = 16
W1 = rng.normal(0, 1.0, (1, H)); b1 = np.zeros(H)
W2 = rng.normal(0, 0.5, (H, 1)); b2 = 0.0
lr = 0.005
for step in range(4000):
    Z = Xin @ W1 + b1; Hd = np.maximum(0, Z)            # ReLU
    yhat = (Hd @ W2 + b2).ravel()
    err = (yhat - y) / x.size
    gW2 = Hd.T @ err[:, None]; gb2 = err.sum()
    dH = (err[:, None] @ W2.T) * (Z > 0)
    gW1 = Xin.T @ dH; gb1 = dH.sum(0)
    W2 -= lr * gW2; b2 -= lr * gb2; W1 -= lr * gW1; b1 -= lr * gb1

Zf = x[:, None] @ W1 + b1; fit = (np.maximum(0, Zf) @ W2 + b2).ravel()
axR.scatter(x, y, s=8, color="#c9ccd1", alpha=0.6)
axR.plot(x, ytrue, color="#1f8a4c", lw=2.0, ls="--", label="true function")
axR.plot(x, fit, color="#1f3a5f", lw=2.2, label=f"{H}-unit ReLU net")
axR.set_title("Learned fit (sum of ReLU hinges)")
axR.set_xlabel("x"); axR.set_ylabel("y"); axR.legend(fontsize=8, frameon=False)
fig.tight_layout()
plt.show()

Training: gradient descent and backpropagation

Training means finding the weights that minimise a loss (penalised least squares for regression, cross-entropy for classification). (James et al. 2021) The machinery: (James et al. 2021)

  • Gradient descent nudges every weight downhill along the loss gradient; the learning rate \eta sets the step size (too big diverges, too small crawls).
  • Stochastic gradient descent (SGD) computes the gradient on small minibatches instead of the whole dataset — essential at scale; one pass through all batches is an epoch.
  • Backpropagation is the efficient chain rule: it propagates the error backward through the layers to get the gradient with respect to every parameter in one sweep. (James et al. 2021)
  • Non-convexity. The loss surface has local minima and saddle points — and remarkably, the noise in SGD’s minibatch sampling helps the optimiser skitter past them. (James et al. 2021)

Regularization: taming a model with too many parameters

A network can have more parameters than observations, so overfitting is the central danger. (James et al. 2021) The countermeasures: (James et al. 2021)

  • Weight decay — an \ell_2/ridge penalty shrinking weights toward zero.
  • Dropout — randomly zero out a fraction of units each training step, forcing redundant, robust pathways rather than fragile co-adaptations. (James et al. 2021)
  • Early stopping — watch a validation set and stop when out-of-sample error starts to rise. (James et al. 2021)
  • Data augmentation — manufacture new training examples by perturbing existing ones (rotate/crop an image), enlarging effective n.

Specialised architectures (conceptually)

Beyond the plain feed-forward net: (James et al. 2021)

  • Convolutional neural networks (CNNs) — for images: weight sharing (the same small filter slides across the image) and pooling tame pixel dimensionality; deep variants like ResNet use skip connections so gradients survive many layers.
  • Recurrent neural networks (RNNs) — for sequences (text, audio): they carry a state across steps, like a nonlinear autoregression; LSTMs stabilise the information flow.
  • Transformers — today’s state of the art for language (BERT, and the architecture behind large language models): self-attention weights every part of a sequence against every other, regardless of distance. (James et al. 2021)

When neural nets actually win

The practical verdict, and the reason this track keeps ML separate: neural nets are not always better. (Jacobucci et al. 2023) They shine on unstructured data — images, text, audio — where features can’t be hand-engineered, and they need very large n to train. (James et al. 2021) On the small, tabular datasets typical of social science, a well-specified linear model or random forest often matches or beats an ad-hoc network — with far more interpretability. (Jacobucci et al. 2023)

NoteThe numbers tell the story
  • Big, unstructured data: in a hedonic price study with 10 million observations, a deep network hit test R^2\approx90\%, beating random forests (80%) and linear models (70%). (James et al. 2021)
  • Small, tabular data: on the Wage data, plain OLS had a test MSE of 0.228 while a simple neural net trailed at 0.249. (James et al. 2021)

Same method, opposite verdicts — set entirely by sample size and data type. Reach for a net when you have millions of unstructured examples; reach for a regression when you have a survey.

Common pitfalls

WarningWhere this goes wrong

  • A neural net on small tabular data. It rarely beats a tuned forest or regression and forfeits interpretability; match the tool to n and data type. (Jacobucci et al. 2023)

  • Skipping regularization. With more parameters than data, weight decay / dropout / early stopping are not optional. (James et al. 2021)

  • A bad learning rate. Too large diverges, too small never converges; it’s the first thing to tune. (James et al. 2021)

  • Reading weights as effects. A network’s millions of weights are uninterpretable and not causal; use interpretability tools, and never confuse them with explanation. (Jacobucci et al. 2023)

  • Believing depth always helps. Past a point it adds parameters and overfitting, not signal; validate. (James et al. 2021)

Appendix · the mathematics

Forward pass (one hidden layer). With inputs Z\in\mathbb R^p, hidden weights A\in\mathbb R^{H\times p}, output weights w: H_m=\sigma\big((AZ)_m+a_m\big),\qquad \hat Y=w^\top H + b . Stacking L such layers, \hat Y is a depth-L composition of (linear \to nonlinear) maps — flexible because of the repeated \sigma.

Backpropagation. For loss \mathcal L, the chain rule gives, layer by layer from the output, \delta^{(L)}=\nabla_{\hat Y}\mathcal L and \delta^{(\ell)}=(W^{(\ell+1)\top}\delta^{(\ell+1)})\odot \sigma'(z^{(\ell)}), with parameter gradients \partial\mathcal L/\partial W^{(\ell)}=\delta^{(\ell)} (h^{(\ell-1)})^\top. One backward sweep yields all gradients in O(\text{#params}) — the efficiency that makes training feasible. For ReLU, \sigma'(z)=\mathbf 1[z>0], so gradients flow only through active units (and the figure’s fit is exactly piecewise-linear).

SGD update. \theta \leftarrow \theta-\eta\,\widehat{\nabla_\theta\mathcal L}, the gradient estimated on a minibatch. The minibatch noise is a feature: it perturbs the trajectory enough to escape sharp local minima and saddle points, often landing in flatter, better-generalising basins.

Connections

  • Builds on: logistic regression (a one-neuron sigmoid/softmax classifier), regularization (weight decay, early stopping), bias–variance (overfitting), and the accuracy–interpretability frontier (the opaque extreme).
  • Leads to: unsupervised learning (autoencoders/embeddings are representation learning without labels) and the social-science workflow of ML for social science.
  • Bridges back: a network is the ultimate predictive black box — superb at \hat Y, mute about why; its weights are not effects.

forest](../04-predictive/trees-and-forests.qmd) often **matches or beats** an ad-hoc network — with far
more interpretability. [@jacobucci_ml]

::: {.callout-note title="The numbers tell the story"}
- **Big, unstructured data:** in a hedonic price study with **10 million** observations, a deep network
  hit **test $R^2\approx90\%$**, beating random forests (**80%**) and linear models (**70%**). [@islr]
- **Small, tabular data:** on the **Wage** data, plain **OLS** had a test MSE of **0.228** while a simple
  neural net trailed at **0.249.** [@islr]

Same method, opposite verdicts — set entirely by sample size and data type. Reach for a net when you have
millions of unstructured examples; reach for a regression when you have a survey.
:::

## Common pitfalls

::: {.callout-warning title="Where this goes wrong"}

- **A neural net on small tabular data.** It rarely beats a tuned forest or regression and forfeits
  interpretability; match the tool to $n$ and data type. [@jacobucci_ml]

- **Skipping regularization.** With more parameters than data, weight decay / dropout / early stopping
  are not optional. [@islr]

- **A bad learning rate.** Too large diverges, too small never converges; it's the first thing to tune.
  [@islr]

- **Reading weights as effects.** A network's millions of weights are uninterpretable and **not causal**;
  use [interpretability tools](../04-predictive/interpretability-and-ml.qmd), and never confuse them with
  explanation. [@jacobucci_ml]

- **Believing depth always helps.** Past a point it adds parameters and overfitting, not signal; validate.
  [@islr]

:::

## Appendix · the mathematics {.unnumbered}

::: {.math-appendix}

**Forward pass (one hidden layer).** With inputs $Z\in\mathbb R^p$, hidden weights $A\in\mathbb
R^{H\times p}$, output weights $w$:
$$
H_m=\sigma\big((AZ)_m+a_m\big),\qquad \hat Y=w^\top H + b .
$$
Stacking $L$ such layers, $\hat Y$ is a depth-$L$ composition of (linear $\to$ nonlinear) maps — flexible
*because* of the repeated $\sigma$.

**Backpropagation.** For loss $\mathcal L$, the chain rule gives, layer by layer from the output,
$\delta^{(L)}=\nabla_{\hat Y}\mathcal L$ and
$\delta^{(\ell)}=(W^{(\ell+1)\top}\delta^{(\ell+1)})\odot \sigma'(z^{(\ell)})$, with parameter gradients
$\partial\mathcal L/\partial W^{(\ell)}=\delta^{(\ell)} (h^{(\ell-1)})^\top$. One backward sweep yields
*all* gradients in $O(\text{#params})$ — the efficiency that makes training feasible. For **ReLU**,
$\sigma'(z)=\mathbf 1[z>0]$, so gradients flow only through *active* units (and the figure's fit is
exactly piecewise-linear).

**SGD update.** $\theta \leftarrow \theta-\eta\,\widehat{\nabla_\theta\mathcal L}$, the gradient estimated
on a minibatch. The minibatch noise is a *feature*: it perturbs the trajectory enough to escape sharp
local minima and saddle points, often landing in flatter, better-generalising basins.

:::
