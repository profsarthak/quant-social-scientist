Gaussian Processes

  • Show All Code
  • Hide All Code
  • View Source

Put a probability distribution not over parameters, but over functions — and get back a prediction that knows exactly where it doesn’t know.

Every model so far fit the parameters of a fixed shape — a line, a tree, a network. A Gaussian process (GP) is different: it places a probability distribution directly over functions, then conditions on the data to get a posterior over functions. The reward is the most honest uncertainty in machine learning — the predictive band pinches tight where you have data and fans out where you don’t — all in closed form, with the smoothness assumptions made explicit through a kernel.

Why it matters

GPs are the Bayesian, nonparametric answer to regression: small-data, uncertainty-critical problems — Bayesian optimisation, spatial statistics, calibration, active learning — where knowing what you don’t know matters as much as the point prediction. They also tie a bow on the ML track: a GP is the Bayesian generalisation of the kernel methods behind SVMs, upgraded from point estimates to full predictive distributions.

A distribution over functions

Where ordinary regression estimates a fixed parameter vector \theta, a GP defines a distribution over functions (Murphy 2012). Its defining property: any finite set of function values \{f(x_1),\dots,f(x_N)\} has a joint multivariate Gaussian distribution (Murphy 2012). Because it assumes no fixed functional form (not linear, not polynomial), it is nonparametric — its effective complexity grows with the data, letting it capture almost any smooth relationship (Murphy 2012).

The two ingredients: mean and kernel

A GP is fully specified by two functions (Murphy 2012):

  • Mean function m(x) = \mathbb{E}[f(x)] — the prior expected value; in practice usually set to 0, since the covariance carries the modelling load (Murphy 2012).
  • Covariance / kernel function k(x,x') — which encodes the central assumption that nearby inputs have correlated outputs (Murphy 2012).

The default is the squared-exponential (RBF) kernel (Murphy 2012):

k(x,x') = \sigma_f^2 \exp\!\Big(-\frac{(x-x')^2}{2\ell^2}\Big).

Its hyperparameters are interpretable: the length-scale \ell controls smoothness — large \ell gives slowly-varying functions, small \ell lets them be “wiggly” — and the signal variance \sigma_f^2 sets the vertical scale (Murphy 2012).

GP regression: the predictive posterior

We observe noisy data y = f(x) + \epsilon, \epsilon\sim\mathcal N(0,\sigma_y^2), which adds a “nugget” to the covariance: K_y = K + \sigma_y^2 I (Murphy 2012). Conditioning the joint Gaussian prior on the data gives a closed-form Gaussian predictive distribution at a new point x_*, \mathcal N(\mu_*,\sigma_*^2), with (Murphy 2012):

\mu_* = k_*^\top K_y^{-1}\,y, \qquad \sigma_*^2 = k(x_*,x_*) - k_*^\top K_y^{-1} k_* ,

where k_* is the vector of kernel values between x_* and the training inputs. The mean is a weighted sum of the observed y’s; the variance is the prior variance minus what the data explains. The signature behaviour: predictive variance is small near the training data and grows as you move into unexplored regions — calibrated uncertainty, for free (Murphy 2012).

Show the GP code 
import numpy as np
import matplotlib.pyplot as plt

rng = np.random.default_rng(3)
def rbf(a, b, ell=1.0, sf=1.0):
    d = a[:, None] - b[None, :]
    return sf**2 * np.exp(-0.5 * (d/ell)**2)

# training data from a smooth latent function + noise (note the gap on the right)
Xtr = np.array([-4.0, -3.0, -1.5, -0.5, 0.5, 1.5, 2.0])
ytr = np.sin(Xtr) + 0.3*Xtr + rng.normal(0, 0.15, Xtr.size)
ell, sf, sy = 1.0, 1.0, 0.15

Xte = np.linspace(-6, 6, 250)
Ky = rbf(Xtr, Xtr, ell, sf) + sy**2 * np.eye(Xtr.size)
Ks = rbf(Xte, Xtr, ell, sf)
Kss = rbf(Xte, Xte, ell, sf)
Ky_inv = np.linalg.inv(Ky)

mu = Ks @ Ky_inv @ ytr
cov = Kss - Ks @ Ky_inv @ Ks.T
sd = np.sqrt(np.clip(np.diag(cov), 0, None))

# three posterior function samples (jitter for numerical PSD)
L = np.linalg.cholesky(cov + 1e-9*np.eye(Xte.size))
samples = mu[:, None] + L @ rng.normal(size=(Xte.size, 3))

fig, ax = plt.subplots(figsize=(7.6, 4.3))
ax.fill_between(Xte, mu - 2*sd, mu + 2*sd, color="#1f4e79", alpha=0.15,
                label="±2 SD predictive band")
for j in range(3):
    ax.plot(Xte, samples[:, j], lw=0.9, alpha=0.7,
            color=["#c0392b", "#1f8a4c", "#d98a29"][j])
ax.plot(Xte, mu, color="#1f4e79", lw=2.4, label="posterior mean")
ax.plot(Xtr, ytr, "o", color="black", ms=6, label="observations")
ax.set_xlabel("x"); ax.set_ylabel("f(x)")
ax.set_title("GP regression: uncertainty pinches at data, fans out away from it")
ax.legend(fontsize=8, frameon=False, loc="upper left"); ax.set_ylim(-4, 4)
fig.tight_layout()
plt.show()

Learning the hyperparameters

The hyperparameters \theta=(\ell,\sigma_f,\sigma_y) are learned by maximising the log marginal likelihood (Murphy 2012):

\log p(y\mid X,\theta) = \underbrace{-\tfrac12\,y^\top K_y^{-1} y}_{\text{data fit}} \ \underbrace{-\ \tfrac12\log|K_y|}_{\text{complexity penalty}}\ -\ \tfrac{N}{2}\log 2\pi .

This is automatic Occam’s razor: the data-fit term rewards models that pass through the points, while the complexity term -\tfrac12\log|K_y| rewards simpler, smoother functions (smaller |K_y|). The marginal likelihood balances them and prevents overfitting without a separate validation set (Murphy 2012) — the Bayesian model-comparison logic of WAIC/evidence, here in closed form.

Kernels, and the O(N^3) wall

GPs are the Bayesian generalisation of kernel methods like SVMs — but where an SVM gives a point prediction, a GP gives a well-calibrated probability distribution (Murphy 2012). The price is computational (Murphy 2012):

  • The bottleneck. Every fit requires inverting (or Cholesky-decomposing) the N\times N kernel matrix — O(N^3) time and O(N^2) memory (Murphy 2012). GPs are a small-/medium-data method.
  • Scaling up. Sparse approximations use M \ll N inducing points to summarise the data, cutting cost to O(M^2 N) — linear in N (Murphy 2012).

Common pitfalls

WarningWhere this goes wrong
  • Trusting the kernel blindly. The kernel is your prior; an RBF assumes smooth, stationary functions — wrong for jumps, periodicity, or trends. Choose (or compose) kernels deliberately. (Murphy 2012)
  • Throwing a GP at big data. O(N^3) makes exact GPs infeasible beyond a few thousand points; use sparse / inducing-point methods. (Murphy 2012)
  • Ignoring the noise term \sigma_y^2. Too small and the GP interpolates noise; too large and it oversmooths. Learn it via the marginal likelihood. (Murphy 2012)
  • Over-reading extrapolation. Far from data the posterior reverts to the prior mean with a wide band — that’s honest, not informative; don’t trust the mean out there. (Murphy 2012)
  • Numerical instability. K_y can be near-singular; add a small jitter to the diagonal and use Cholesky rather than a raw inverse. (Murphy 2012)

Appendix · the mathematics

Conditioning a joint Gaussian (the whole trick). Stack training outputs y and a test value f_*. Under the GP prior they are jointly Gaussian,

\begin{bmatrix} y \\ f_* \end{bmatrix} \sim \mathcal N\!\left(0,\ \begin{bmatrix} K_y & k_* \\ k_*^\top & k(x_*,x_*) \end{bmatrix}\right).

The standard Gaussian conditioning formula p(f_*\mid y) then gives the predictive mean \mu_* = k_*^\top K_y^{-1} y and variance \sigma_*^2 = k(x_*,x_*) - k_*^\top K_y^{-1} k_* directly — no optimisation, no sampling. All of GP regression is one application of “condition the joint Normal.”

Why the variance ignores y. \sigma_*^2 depends only on the inputs X, x_* (through the kernel), not the observed y — the GP’s uncertainty is set by where you have data, not what value it took. Hence the band’s shape is fixed by the sampling design.

The complexity penalty. -\tfrac12\log|K_y| is large (more negative) when K_y has big eigenvalues — i.e. when the kernel permits very flexible functions. Maximising the marginal likelihood thus trades fit against this volume term, the GP’s built-in Occam factor.

Connections

  • Builds on: support vector machines and the kernel trick (GPs are their Bayesian cousin), common distributions (the multivariate Normal), and regularization (the complexity penalty).
  • Echoes: Bayesian model comparison — the marginal likelihood is the same automatic-Occam machinery.
  • Contrasts with: trees & forests — non-smooth, scalable, but without the GP’s calibrated, closed-form uncertainty.

$\mu_* = k_*^\top K_y^{-1} y$ and variance $\sigma_*^2 = k(x_*,x_*) - k_*^\top K_y^{-1} k_*$ directly — no
optimisation, no sampling. All of GP regression is one application of "condition the joint Normal."

**Why the variance ignores $y$.** $\sigma_*^2$ depends only on the *inputs* $X, x_*$ (through the kernel),
not the observed $y$ — the GP's uncertainty is set by **where** you have data, not what value it took. Hence
the band's shape is fixed by the sampling design.

**The complexity penalty.** $-\tfrac12\log|K_y|$ is large (more negative) when $K_y$ has big eigenvalues —
i.e. when the kernel permits very flexible functions. Maximising the marginal likelihood thus trades fit
against this volume term, the GP's built-in [Occam factor](../02-inference/bayesian-model-comparison.qmd).

:::
