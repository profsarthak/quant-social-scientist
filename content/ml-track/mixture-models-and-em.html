Mixture Models & the EM Algorithm

  • Show All Code
  • Hide All Code
  • View Source

k-means draws hard borders and guesses round blobs. Its probabilistic parent — the Gaussian mixture, fit by EM — gives soft memberships, real densities, and one of the most reusable algorithms in statistics.

k-means clustering forces every point into exactly one group and assumes spherical clusters. The Gaussian mixture model (GMM) is its principled generalisation: the data are a weighted blend of Gaussians, each point gets a soft membership (a probability of belonging to each cluster), and the clusters can be elliptical. Fitting it requires the EM algorithm — an elegant alternation between guessing the hidden labels and re-estimating the parameters — that turns out to solve a vast class of latent-variable and missing-data problems far beyond clustering.

Why it matters

Mixtures are the natural model whenever a population is a blend of unseen subgroups — latent classes of voters, patient phenotypes, topics in text. And EM is the workhorse behind them, behind factor analysis, behind hidden Markov models, and behind principled missing-data handling. Understanding EM once unlocks all of them.

The Gaussian mixture model

A mixture model represents a population as a weighted sum of K base distributions (Murphy 2012). In a GMM each component k is a multivariate Normal with its own mean \mu_k and covariance \Sigma_k (Murphy 2012). A discrete latent variable z_i \in \{1,\dots,K\} records each point’s hidden cluster assignment, with prior the mixing weights \pi_k (0\le\pi_k\le1, \sum_k\pi_k=1) (Murphy 2012). The density of an observation is the convex combination (Murphy 2012):

p(x_i\mid\theta) = \sum_{k=1}^{K} \pi_k\,\mathcal N(x_i\mid\mu_k,\Sigma_k).

The payoff over k-means is soft clustering: rather than a hard label, the GMM gives the posterior probability p(z_i=k\mid x_i,\theta) that point i belongs to cluster k — its responsibility r_{ik} (Murphy 2012).

Why direct maximum likelihood is hard

You’d like to just maximise the log-likelihood, but the latent z blocks you (Murphy 2012):

\ell(\theta) = \sum_{i=1}^{N}\log\!\Big[\sum_{z_i} p(x_i,z_i\mid\theta)\Big].

The log cannot be pushed inside the sum over the hidden labels, so the terms won’t separate (Murphy 2012). Worse, the surface is non-convex and multimodal, with label-switching unidentifiability — permuting the cluster labels gives the identical likelihood (Murphy 2012). Direct optimisation is a mess.

The EM algorithm

EM sidesteps the log-sum by alternating two steps that each have closed forms, climbing to a local optimum (Murphy 2012).

E-step (Expectation). Using the current parameters, compute the responsibilities — the posterior membership of every point in every cluster (Murphy 2012):

r_{ik} = \frac{\pi_k\,\mathcal N(x_i\mid\mu_k,\Sigma_k)}{\sum_{k'=1}^{K}\pi_{k'}\,\mathcal N(x_i\mid\mu_{k'},\Sigma_{k'})}.

M-step (Maximisation). Update each parameter as a responsibility-weighted MLE, with r_k=\sum_i r_{ik} the soft count in cluster k (Murphy 2012):

\pi_k = \frac{r_k}{N},\qquad \mu_k = \frac{\sum_i r_{ik}\,x_i}{r_k},\qquad \Sigma_k = \frac{\sum_i r_{ik}\,(x_i-\mu_k)(x_i-\mu_k)^\top}{r_k}.

Two guarantees and one caveat (Murphy 2012):

  • Monotonicity. Every iteration increases (or holds) the observed-data log-likelihood — EM never goes downhill.
  • Local optima. It converges to a local optimum or saddle, so initialisation matters; use multiple random restarts or initialise from k-means.
Show the EM code 
import numpy as np
import matplotlib.pyplot as plt

rng = np.random.default_rng(1)
# true 1-D mixture: three Gaussians
N = 900
z = rng.choice(3, size=N, p=[0.3, 0.45, 0.25])
mus_true = np.array([-3.0, 1.0, 5.0]); sds_true = np.array([1.0, 0.8, 1.2])
x = rng.normal(mus_true[z], sds_true[z])

def normpdf(x, m, s):
    return np.exp(-0.5*((x-m)/s)**2) / (s*np.sqrt(2*np.pi))

K = 3
mu = np.quantile(x, [0.2, 0.5, 0.8]).astype(float)   # init
sd = np.full(K, x.std()); pi = np.full(K, 1/K)
loglik = []
for it in range(25):
    # E-step: responsibilities (N x K)
    comp = np.stack([pi[k]*normpdf(x, mu[k], sd[k]) for k in range(K)], axis=1)
    tot = comp.sum(axis=1)
    loglik.append(np.sum(np.log(tot)))
    r = comp / tot[:, None]
    # M-step: weighted MLEs
    rk = r.sum(axis=0)
    pi = rk / N
    mu = (r * x[:, None]).sum(axis=0) / rk
    sd = np.sqrt((r * (x[:, None] - mu)**2).sum(axis=0) / rk)

fig, (axL, axR) = plt.subplots(1, 2, figsize=(9, 3.8),
                               gridspec_kw={"width_ratios": [1.6, 1]})
grid = np.linspace(x.min()-1, x.max()+1, 400)
axL.hist(x, bins=45, density=True, color="#dde4ea", edgecolor="white")
colors = ["#1f4e79", "#c0392b", "#1f8a4c"]
total = np.zeros_like(grid)
for k in range(K):
    dens = pi[k]*normpdf(grid, mu[k], sd[k]); total += dens
    axL.plot(grid, dens, "--", color=colors[k], lw=1.6)
axL.plot(grid, total, color="#222", lw=2.2, label="fitted mixture")
axL.set_title("GMM fit (3 components)"); axL.set_xlabel("x"); axL.legend(fontsize=8, frameon=False)

axR.plot(range(1, len(loglik)+1), loglik, "o-", color="#1f4e79", ms=3.5, lw=1.6)
axR.set_title("Log-likelihood ↑ every step"); axR.set_xlabel("EM iteration")
axR.set_ylabel("observed-data log-likelihood")
fig.tight_layout()
plt.show()

k-means is “hard EM”

Murphy frames k-means as a “hard EM” special case of GMM-EM (Murphy 2012). Assume equal mixing weights (\pi_k=1/K) and fixed spherical covariances (\Sigma_k=\sigma^2 I); then take the small-variance limit \sigma^2\to0 (Murphy 2012). The soft responsibilities collapse to hard 0/1 assignments (each point to its nearest centroid), and the two steps become exactly k-means: the E-step assigns points to the nearest centre, the M-step recomputes centres as the mean of assigned points (Murphy 2012). k-means is a GMM with the uncertainty switched off.

EM beyond mixtures

EM is far more general than clustering (Murphy 2012):

  • Missing data — e.g. fitting a multivariate Normal with missing entries (the missing values are the latent variables).
  • Latent-variable models — factor analysis and, for sequences, hidden Markov models (where EM is the Baum–Welch algorithm).

Choosing K is a model-selection problem (Murphy 2012). Unlike k-means reconstruction error (which always falls as K grows), a GMM’s validation negative log-likelihood is U-shaped — it eventually rises as the model overfits — so you can pick K at the minimum, or use BIC / variational Bayes to penalise complexity (Murphy 2012).

Common pitfalls

WarningWhere this goes wrong
  • Trusting a single EM run. It finds a local optimum; use several random restarts (or a k-means init) and keep the best log-likelihood. (Murphy 2012)
  • Variance collapse. A component can latch onto one point and shrink its variance toward zero, sending the likelihood to infinity — regularise the covariance (a small ridge) or use priors. (Murphy 2012)
  • Picking K by training likelihood. It only ever improves with K; judge on a validation set or by BIC. (Murphy 2012)
  • Forgetting label-switching. Cluster labels are arbitrary; don’t over-interpret which index is which across runs. (Murphy 2012)
  • Using a GMM when clusters aren’t roughly Gaussian. The model imposes elliptical components; very non-Gaussian shapes need a different density. (Murphy 2012)

NoteWho says what

Appendix · the mathematics

Why EM climbs the likelihood. EM maximises a lower bound. For any distribution q(z) over the latent labels, Jensen’s inequality gives \ell(\theta) \ge \sum_i \mathbb{E}_{q}[\log p(x_i,z_i\mid\theta)] + H(q) \equiv \mathcal L(q,\theta), the ELBO. The E-step sets q(z_i)=p(z_i\mid x_i,\theta^{\text{old}}) (the responsibilities), which makes the bound tight (\mathcal L=\ell). The M-step maximises \mathcal L over \theta. Since the E-step touches the bound to \ell and the M-step only raises it, \ell cannot decrease — the monotonicity guarantee.

The M-step updates are weighted MLEs. Maximising the expected complete-data log-likelihood \sum_i\sum_k r_{ik}\big[\log\pi_k + \log\mathcal N(x_i\mid\mu_k,\Sigma_k)\big] over each parameter (with the \sum_k\pi_k=1 Lagrange constraint) yields exactly the soft-count formulas above — every point contributes to every cluster, weighted by its responsibility.

The k-means limit. With \Sigma_k=\sigma^2 I, the responsibility r_{ik}\propto \exp(-\|x_i-\mu_k\|^2/2\sigma^2); as \sigma^2\to0 the softmax becomes an argmax, so r_{ik}\in\{0,1\} picks the nearest centroid — the hard E-step of k-means.

Connections

  • Builds on: unsupervised learning / k-means (the hard-clustering special case), common distributions (the Gaussian components), and maximum likelihood (what the M-step maximises).
  • Generalises to: factor analysis and hidden Markov models, both fit by EM.
  • Compares with: hierarchical Bayesian models — a fully Bayesian mixture puts priors on \pi,\mu,\Sigma and samples instead of point-maximising.

## Common pitfalls

::: {.callout-warning title="Where this goes wrong"}
- **Trusting a single EM run.** It finds a *local* optimum; use several random restarts (or a k-means init)
  and keep the best log-likelihood. [@murphy_ml]
- **Variance collapse.** A component can latch onto one point and shrink its variance toward zero, sending
  the likelihood to infinity — regularise the covariance (a small ridge) or use priors. [@murphy_ml]
- **Picking K by training likelihood.** It only ever improves with K; judge on a validation set or by BIC.
  [@murphy_ml]
- **Forgetting label-switching.** Cluster labels are arbitrary; don't over-interpret which index is which
  across runs. [@murphy_ml]
- **Using a GMM when clusters aren't roughly Gaussian.** The model imposes elliptical components; very
  non-Gaussian shapes need a different density. [@murphy_ml]
:::

## Appendix · the mathematics {.unnumbered}

::: {.math-appendix}

**Why EM climbs the likelihood.** EM maximises a lower bound. For any distribution $q(z)$ over the latent
labels, Jensen's inequality gives
$\ell(\theta) \ge \sum_i \mathbb{E}_{q}[\log p(x_i,z_i\mid\theta)] + H(q) \equiv \mathcal L(q,\theta)$, the
**ELBO**. The **E-step** sets $q(z_i)=p(z_i\mid x_i,\theta^{\text{old}})$ (the responsibilities), which makes
the bound **tight** ($\mathcal L=\ell$). The **M-step** maximises $\mathcal L$ over $\theta$. Since the
E-step touches the bound to $\ell$ and the M-step only raises it, $\ell$ **cannot decrease** — the
monotonicity guarantee.

**The M-step updates are weighted MLEs.** Maximising the expected complete-data log-likelihood
$\sum_i\sum_k r_{ik}\big[\log\pi_k + \log\mathcal N(x_i\mid\mu_k,\Sigma_k)\big]$ over each parameter (with the
$\sum_k\pi_k=1$ Lagrange constraint) yields exactly the soft-count formulas above — every point contributes
to every cluster, weighted by its responsibility.

**The k-means limit.** With $\Sigma_k=\sigma^2 I$, the responsibility
$r_{ik}\propto \exp(-\|x_i-\mu_k\|^2/2\sigma^2)$; as $\sigma^2\to0$ the softmax becomes an argmax, so
$r_{ik}\in\{0,1\}$ picks the nearest centroid — the hard E-step of k-means.

:::
