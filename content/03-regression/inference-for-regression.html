Inference for Regression Coefficients

  • Show All Code
  • Hide All Code
  • View Source

A coefficient is an estimate — so attach a standard error, test it, and bound it, before you believe it.

A regression coefficient is a statistic, so it has a sampling distribution: run the study again and it would land somewhere slightly different. Everything in this entry — the standard error, the t-test, the confidence interval, the F-test — is the Part II inference machinery attached to \hat\beta. The discipline is the same: never report a coefficient without its uncertainty.

Why it matters

The printout says the effect of education on log-wage is 0.092. Is that real, or could a fresh sample have produced 0? You answer with the coefficient’s standard error and the tests built on it. (Wooldridge 2019) This is where regression stops being description and becomes inference: every “significant effect,” every “we cannot reject,” every error bar in an empirical paper is one of the procedures below. And the statistical-vs-practical distinction returns with a vengeance — with enough data, everything is significant.

The sampling distribution of a coefficient

To do exact inference at any sample size, Wooldridge adds one assumption to the Gauss–Markov five: (Wooldridge 2019)

MLR.6 (Normality): the error u is independent of the regressors and distributed \mathcal{N}(0, \sigma^2).

MLR.1–MLR.6 are the Classical Linear Model. Under them, each OLS estimator is Normally distributed around the truth: (Wooldridge 2019)

\hat\beta_j \sim \mathcal{N}\!\big(\beta_j,\ \operatorname{Var}(\hat\beta_j)\big).

(Even without MLR.6, the CLT delivers approximate Normality in large samples — which is why regression inference works so widely.) The standard error \operatorname{se}(\hat\beta_j) is the estimated standard deviation of this distribution, built from the variance formula with \sigma^2 replaced by its estimate \hat\sigma^2 (the SER squared). (Wooldridge 2019) The standard error is the whole ball game: it is what converts a point estimate into a claim.

Testing a single coefficient: the t-test

To ask “could the true effect be zero?”, test H_0:\beta_j = 0 with the t-statistic — the estimate measured in standard errors: (Wooldridge 2019)

t = \frac{\hat\beta_j}{\operatorname{se}(\hat\beta_j)}.

If |t| exceeds the critical value from the t-distribution with n-k-1 degrees of freedom, reject H_0 — the coefficient is statistically significant. (Wooldridge 2019) The companion confidence interval gives the range of values you can’t reject: (Wooldridge 2019)

\hat\beta_j \pm c\cdot\operatorname{se}(\hat\beta_j),

with c \approx 2 for 95%. The interval and the test are duals: the coefficient is significant at 5% exactly when its 95% interval excludes zero.

NoteWooldridge’s WAGE1 t-test

In the log-wage regression (n=526), the education coefficient is \hat\beta_{\text{educ}} = 0.092 with \operatorname{se} = 0.007. So t = 0.092/0.007 \approx 13.1 — vastly beyond any critical value. Education is overwhelmingly statistically significant: a sampling fluke essentially could not produce a t of 13. (Wooldridge 2019) (The 95% interval, 0.092 \pm 2(0.007) = [0.078, 0.106], nowhere near zero.)

Significance is estimate ÷ standard error

The t-statistic makes vivid that significance is not about the size of an effect but about its size relative to its noise. Below, two coefficients have the identical estimate (0.5); only their standard errors differ. The precise one is clearly significant; the noisy one can’t be distinguished from zero — same number, opposite verdict.

Show the figure code 
import numpy as np
import matplotlib.pyplot as plt
from statistics import NormalDist

est = 0.5
xs = np.linspace(-1.0, 2.0, 600)

fig, ax = plt.subplots(figsize=(7, 3.7))
for se, color, label in [(0.10, "#1f3a5f", "se = 0.10  (t = 5.0, significant)"),
                         (0.35, "#d98a29", "se = 0.35  (t = 1.4, not significant)")]:
    nd = NormalDist(est, se)
    ax.plot(xs, [nd.pdf(v) for v in xs], color=color, lw=2.0, label=label)
    lo, hi = est - 1.96 * se, est + 1.96 * se
    y = -0.25 if se == 0.10 else -0.5
    ax.errorbar(est, y, xerr=1.96 * se, fmt="o", color=color, capsize=4, lw=2)
ax.axvline(0, color="#c0392b", ls="--", lw=1.6)
ax.text(0.02, ax.get_ylim()[1] * 0.85, "H₀: β = 0", color="#c0392b", fontsize=9)
ax.set_yticks([]); ax.set_xlabel("coefficient value")
ax.set_title("Same estimate (0.5), different precision, different verdict")
ax.legend(fontsize=8, frameon=False, loc="upper right")
fig.tight_layout()
plt.show()

Testing several coefficients at once: the F-test

Some questions involve groups of coefficients: “do any of these four neighbourhood variables matter?”, “should this block of regressors be in the model at all?” Testing them one t-test at a time invites the multiple-comparisons problem; the F-test asks the joint question in one shot. (Wooldridge 2019) It compares a restricted model (the suspect coefficients forced to zero) against the unrestricted model (all included), via their residual sums of squares: (Wooldridge 2019)

F = \frac{(\text{SSR}_r - \text{SSR}_{ur})/q}{\text{SSR}_{ur}/(n-k-1)},

where q is the number of restrictions. The logic is pure ANOVA: does dropping those regressors make the fit significantly worse? A special case — the test of overall significance — restricts all slopes to zero (H_0:\beta_1=\cdots=\beta_k=0), asking whether the model explains anything at all. (Wooldridge 2019) (This F is what reproduces the ANOVA F-test when the regressors are group dummies — regression contains ANOVA.)

Statistical vs practical significance — again

Wooldridge hammers the distinction, because regression makes it so easy to confuse: (Wooldridge 2019)

  • Statistical significance (the t-stat / p-value) asks whether the effect is distinguishable from zero — a question about noise.
  • Practical (economic) significance (the magnitude of \hat\beta_j) asks whether the effect is big enough to matter — a question about substance.

With a huge sample, a trivially small coefficient can be “highly significant” (t enormous, effect meaningless); with a small sample, a large, important effect can be “insignificant.” (Wooldridge 2019) Always report and interpret the coefficient’s size in real units, not just its stars. A significant 0.2%-per-year effect and a significant 8%-per-year effect are both “significant” and worlds apart.

Prediction: two kinds of interval

Inference isn’t only about coefficients — often the goal is to predict y for a new case with regressor values x_0. The point prediction is just the fitted value, \hat y_0 = \hat\beta_0 + \hat\beta_1 x_{0}+\cdots. (Wooldridge 2019) But “how uncertain is that prediction?” has two different answers, and confusing them is a classic error:

  • A confidence interval for the mean, E[y\mid x_0] — the average outcome for everyone with characteristics x_0. Its only uncertainty comes from the estimated coefficients, so it is narrow and shrinks toward zero as n\to\infty.
  • A prediction interval for a single new individual y_0 — that person’s actual outcome. It carries the coefficient uncertainty plus the irreducible scatter of individuals around the line, the error variance \sigma^2. So it is much wider and does not vanish as n grows: even with infinite data and a perfectly known line, individuals still scatter by \sigma. (Wooldridge 2019)
Show the figure code 
import numpy as np
import matplotlib.pyplot as plt
from statistics import NormalDist

rng = np.random.default_rng(2)
n = 60
x = rng.uniform(0, 10, n)
sigma = 2.5
y = 1.0 + 0.8 * x + rng.normal(0, sigma, n)

xbar, Sxx = x.mean(), np.sum((x - x.mean()) ** 2)
b1 = np.cov(x, y, bias=True)[0, 1] / np.var(x)
b0 = y.mean() - b1 * xbar
resid = y - (b0 + b1 * x)
s = np.sqrt(np.sum(resid ** 2) / (n - 2))            # estimated sigma
tcrit = 1.96

xs = np.linspace(0, 10, 100)
se_mean = s * np.sqrt(1 / n + (xs - xbar) ** 2 / Sxx)        # SE of the mean prediction
se_pred = s * np.sqrt(1 + 1 / n + (xs - xbar) ** 2 / Sxx)    # + 1 for an individual
fit = b0 + b1 * xs

fig, ax = plt.subplots(figsize=(7, 4.2))
ax.scatter(x, y, s=16, color="#9aa0a6", alpha=0.6, zorder=2)
ax.fill_between(xs, fit - tcrit * se_pred, fit + tcrit * se_pred,
                color="#d98a29", alpha=0.18, label="95% prediction interval (an individual)")
ax.fill_between(xs, fit - tcrit * se_mean, fit + tcrit * se_mean,
                color="#1f3a5f", alpha=0.35, label="95% confidence interval (the mean)")
ax.plot(xs, fit, color="#1f3a5f", lw=2.2)
ax.set_xlabel("x"); ax.set_ylabel("y")
ax.set_title("Confidence interval (mean) vs prediction interval (individual)")
ax.legend(fontsize=8, frameon=False, loc="upper left")
fig.tight_layout()
plt.show()

The practical upshot: a regression can pin down an average tightly while still being nearly useless for forecasting any single outcome — the prediction band is wide because people are not their average. (For log-outcome models, predicting the level needs a retransformation correction, since \exp of the predicted log under-predicts the mean — the Jensen gap again.)

Common pitfalls

WarningWhere this goes wrong

  • Reading stars instead of magnitudes. Significance is not importance. A tiny, precisely-estimated effect is significant and may be useless; report the size and a confidence interval. (Wooldridge 2019)

  • Mining many t-tests for the significant one. Each coefficient at 5% has a 1-in-20 false-positive rate; with many regressors, use the F-test for joint questions and pre-specify hypotheses. (Wooldridge 2019)

  • Trusting standard errors when MLR.5 fails. Heteroskedasticity or clustering makes the printed SEs wrong (the coefficients are still fine). Use robust/clustered SEs. (Wooldridge 2019)

  • Interpreting a wide interval around zero as ‘no effect.’ It may mean underpowered, not null — the power lesson, in regression dress.

  • Forgetting inference assumes a valid model. A perfect t-statistic on an endogenous coefficient precisely estimates a biased number. Inference quantifies sampling noise, not specification error.

Appendix · the mathematics

From Normal to t. Under the CLM, (\hat\beta_j-\beta_j)/\operatorname{sd}(\hat\beta_j) is exactly standard Normal. Replacing the unknown \sigma in \operatorname{sd} with the estimate \hat\sigma inflates the tails to a t distribution with n-k-1 degrees of freedom — the same \sigma-is-estimated penalty as the one-sample t-test, now with k+1 parameters consumed instead of one.

The F as a fit comparison. Adding regressors can only lower (never raise) the SSR, so \text{SSR}_r \ge \text{SSR}_{ur}. The numerator (\text{SSR}_r-\text{SSR}_{ur})/q is the average improvement per restriction; the denominator \text{SSR}_{ur}/(n-k-1) estimates \sigma^2. Under H_0 both estimate the same noise, so F\approx1; a real contribution makes F large. F\sim F_{q,\,n-k-1} under the null.

t and F coincide for one restriction. For q=1, F = t^2 exactly — the joint test of a single coefficient is just the square of its t-test.

An R²-form of the overall F. Dividing through by SST, the overall-significance F equals \dfrac{R^2/k}{(1-R^2)/(n-k-1)} — large when the model explains a big share of variance relative to its degrees of freedom.

$$

where $q$ is the number of restrictions. The logic is pure
[ANOVA](../02-inference/comparing-groups.qmd): does dropping those regressors make the fit
*significantly* worse? A special case — the **test of overall significance** — restricts *all* slopes
to zero ($H_0:\beta_1=\cdots=\beta_k=0$), asking whether the model explains anything at all.
[@wooldridge_intro] (This F is what reproduces the [ANOVA F-test](../02-inference/comparing-groups.qmd)
when the regressors are group dummies — regression *contains* ANOVA.)

## Statistical vs practical significance — again

Wooldridge hammers the distinction, because regression makes it so easy to confuse:
[@wooldridge_intro]

- **Statistical significance** (the t-stat / p-value) asks whether the effect is distinguishable from
  zero — a question about *noise*.
- **Practical (economic) significance** (the *magnitude* of $\hat\beta_j$) asks whether the effect is
  *big enough to matter* — a question about *substance*.

With a huge sample, a trivially small coefficient can be "highly significant" ($t$ enormous, effect
meaningless); with a small sample, a large, important effect can be "insignificant."
[@wooldridge_intro] Always report and interpret the **coefficient's size in real units**, not just its
stars. A significant 0.2%-per-year effect and a significant 8%-per-year effect are both "significant"
and worlds apart.

## Prediction: two kinds of interval

Inference isn't only about coefficients — often the goal is to **predict $y$ for a new case** with
regressor values $x_0$. The point prediction is just the fitted value, $\hat y_0 = \hat\beta_0 +
\hat\beta_1 x_{0}+\cdots$. [@wooldridge_intro] But "how uncertain is that prediction?" has **two
different answers**, and confusing them is a classic error:

- A **confidence interval for the mean**, $E[y\mid x_0]$ — the *average* outcome for everyone with
  characteristics $x_0$. Its only uncertainty comes from the estimated coefficients, so it is
  **narrow** and shrinks toward zero as $n\to\infty$.
- A **prediction interval for a single new individual** $y_0$ — that person's *actual* outcome. It
  carries the coefficient uncertainty **plus the irreducible scatter of individuals around the line**,
  the error variance $\sigma^2$. So it is **much wider** and does **not** vanish as $n$ grows: even with
  infinite data and a perfectly known line, individuals still scatter by $\sigma$. [@wooldridge_intro]

```{python}
#| label: fig-prediction-bands
#| fig-cap: "Two intervals, two questions. The inner blue band is the 95% confidence interval for the mean outcome E[y|x] — narrow, and narrowest near the centre of the data. The outer band is the 95% prediction interval for a single new individual: far wider because it adds the irreducible person-to-person scatter (σ), which no amount of data removes. Predicting an average is easy; predicting a person is not."
#| code-fold: true
#| code-summary: "Show the figure code"

import numpy as np
import matplotlib.pyplot as plt
from statistics import NormalDist

rng = np.random.default_rng(2)
n = 60
x = rng.uniform(0, 10, n)
sigma = 2.5
y = 1.0 + 0.8 * x + rng.normal(0, sigma, n)

xbar, Sxx = x.mean(), np.sum((x - x.mean()) ** 2)
b1 = np.cov(x, y, bias=True)[0, 1] / np.var(x)
b0 = y.mean() - b1 * xbar
resid = y - (b0 + b1 * x)
s = np.sqrt(np.sum(resid ** 2) / (n - 2))            # estimated sigma
tcrit = 1.96

xs = np.linspace(0, 10, 100)
se_mean = s * np.sqrt(1 / n + (xs - xbar) ** 2 / Sxx)        # SE of the mean prediction
se_pred = s * np.sqrt(1 + 1 / n + (xs - xbar) ** 2 / Sxx)    # + 1 for an individual
fit = b0 + b1 * xs

fig, ax = plt.subplots(figsize=(7, 4.2))
ax.scatter(x, y, s=16, color="#9aa0a6", alpha=0.6, zorder=2)
ax.fill_between(xs, fit - tcrit * se_pred, fit + tcrit * se_pred,
                color="#d98a29", alpha=0.18, label="95% prediction interval (an individual)")
ax.fill_between(xs, fit - tcrit * se_mean, fit + tcrit * se_mean,
                color="#1f3a5f", alpha=0.35, label="95% confidence interval (the mean)")
ax.plot(xs, fit, color="#1f3a5f", lw=2.2)
ax.set_xlabel("x"); ax.set_ylabel("y")
ax.set_title("Confidence interval (mean) vs prediction interval (individual)")
ax.legend(fontsize=8, frameon=False, loc="upper left")
fig.tight_layout()
plt.show()
```

The practical upshot: a regression can pin down an **average** tightly while still being nearly useless
for forecasting any **single** outcome — the prediction band is wide because people are not their
average. (For [log-outcome models](functional-form.qmd), predicting the level needs a retransformation
correction, since $\exp$ of the predicted log under-predicts the mean — the
[Jensen](../01-foundations/expectation-and-variance.qmd) gap again.)

## Common pitfalls

::: {.callout-warning title="Where this goes wrong"}

- **Reading stars instead of magnitudes.** Significance is not importance. A tiny, precisely-estimated
  effect is significant and may be useless; report the size and a confidence interval.
  [@wooldridge_intro]

- **Mining many t-tests for the significant one.** Each coefficient at 5% has a 1-in-20 false-positive
  rate; with many regressors, use the [F-test](#testing-several-coefficients-at-once-the-f-test) for
  joint questions and pre-specify hypotheses. [@wooldridge_intro]

- **Trusting standard errors when MLR.5 fails.** Heteroskedasticity or clustering makes the printed
  SEs wrong (the coefficients are still fine). Use [robust/clustered SEs](regression-diagnostics.qmd).
  [@wooldridge_intro]

- **Interpreting a wide interval around zero as 'no effect.'** It may mean *underpowered*, not *null*
  — the [power](../02-inference/statistical-power.qmd) lesson, in regression dress.

- **Forgetting inference assumes a valid model.** A perfect t-statistic on an
  [endogenous](gauss-markov-assumptions.qmd) coefficient precisely estimates a *biased* number.
  Inference quantifies sampling noise, not specification error.

:::

## Appendix · the mathematics {.unnumbered}

::: {.math-appendix}

**From Normal to t.** Under the CLM, $(\hat\beta_j-\beta_j)/\operatorname{sd}(\hat\beta_j)$ is exactly
standard Normal. Replacing the unknown $\sigma$ in $\operatorname{sd}$ with the estimate $\hat\sigma$
inflates the tails to a $t$ distribution with $n-k-1$ degrees of freedom — the same
[$\sigma$-is-estimated penalty](../02-inference/confidence-intervals.qmd) as the one-sample t-test,
now with $k+1$ parameters consumed instead of one.

**The F as a fit comparison.** Adding regressors can only lower (never raise) the SSR, so
$\text{SSR}_r \ge \text{SSR}_{ur}$. The numerator $(\text{SSR}_r-\text{SSR}_{ur})/q$ is the *average
improvement per restriction*; the denominator $\text{SSR}_{ur}/(n-k-1)$ estimates $\sigma^2$. Under
$H_0$ both estimate the same noise, so $F\approx1$; a real contribution makes $F$ large.
$F\sim F_{q,\,n-k-1}$ under the null.

**t and F coincide for one restriction.** For $q=1$, $F = t^2$ exactly — the joint test of a single
coefficient is just the square of its t-test.

**An R²-form of the overall F.** Dividing through by SST, the overall-significance F equals
$\dfrac{R^2/k}{(1-R^2)/(n-k-1)}$ — large when the model explains a big share of variance relative to
its degrees of freedom.

:::
