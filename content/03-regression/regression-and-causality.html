What Regression Can — and Can’t — Tell You

  • Show All Code
  • Hide All Code
  • View Source

The mature verdict: a coefficient is an effect only when you can defend the comparison behind it.

You can now fit, interpret, and stress-test a regression. The last and hardest question is the one that matters most: when is a coefficient a cause, not just a correlation with controls? The honest answer is humbling — almost never automatically. Regression estimates a conditional average; turning that into a causal effect takes an assumption that data alone can’t verify, and usually a research design. This entry is the bridge to the summit.

Why it matters

It is dangerously easy to run a regression, add some controls, and announce “the effect of X on Y.” But holding things constant only neutralises the confounders you observed and included. (Angrist and Pischke 2009) The unobserved ones — ability, motivation, health, a thousand others — sit in the error term, and if they correlate with your regressor, the coefficient is biased no matter how clean the output looks. This entry assembles the threats into one clear-eyed account of regression’s causal limits, and explains why credible causal claims need more than a longer control list — they need the methods of Part V.

The one condition: exogeneity

Strip everything away and a single condition decides whether a slope is causal: the regressor must be exogenous, E[u\mid x]=0 — equivalently, treatment is as good as randomly assigned once you condition on the controls (the conditional independence or selection-on-observables assumption). (Angrist and Pischke 2009; Wooldridge 2019) When it holds, the regression line is the causal conditional expectation function. When it fails, regression still faithfully estimates the best linear approximation to the CEF — but the CEF is descriptive, not causal. (Angrist and Pischke 2009) Regression is an extraordinary describing machine; whether its description is also an explanation is a question about the world, not the math.

Threat 1 — omitted variables (the central one)

The dominant threat, revisited from multiple regression: leave out a confounder and its influence loads onto your coefficient. The logic is the “short = long + bias” identity — the coefficient you get omitting a control equals the true effect plus (\text{effect of omitted on }y)\times(\text{its relation to the included regressor}). (Angrist and Pischke 2009) The classic case is ability bias: regress wages on education without ability, and since ability raises wages and correlates with schooling, the education coefficient is biased upward. (Angrist and Pischke 2009)

The uncomfortable truth: you can never be certain you’ve controlled for everything. (Angrist and Pischke 2009) There is no test that confirms exogeneity; the unobserved confounder is, by definition, unobserved. This is why “I controlled for a lot of stuff” is an argument, not a proof — and why more controls buy credibility, never certainty.

Threat 2 — bad controls (when a control hurts)

The instinct “control for everything” is wrong, and dangerously so. (Angrist and Pischke 2009) A bad control is a variable that is itself an outcome of the treatment — a post-treatment variable. Conditioning on it doesn’t remove confounding; it introduces bias. (Angrist and Pischke 2009)

NoteOccupation, the textbook bad control

Study the effect of education on wages, and it’s tempting to control for occupation. Don’t: education causes occupation (more schooling opens better jobs), so holding occupation fixed closes off one of the very channels through which education raises wages. You’d be measuring only the within-occupation effect and throwing away the part that flows through getting a better job — biasing the total effect, usually toward zero. (Angrist and Pischke 2009) The rule: good controls are predetermined (fixed before treatment); post-treatment variables are bad controls.

Collider bias: how conditioning creates correlation

The subtlest version: conditioning on a common consequence of two variables manufactures an association that isn’t there. This is collider (or selection) bias. (Angrist and Pischke 2009) The picture below is the canonical example: talent and beauty are uncorrelated in the population, but among people selected on their sum (say, working actors, hired for one or the other), they appear negatively correlated — condition on the collider and a spurious relationship springs into being.

Show the figure code 
import numpy as np
import matplotlib.pyplot as plt

rng = np.random.default_rng(4)
n = 2000
talent = rng.normal(0, 1, n)
beauty = rng.normal(0, 1, n)          # independent of talent in the population
hired = (talent + beauty) > 1.6       # selected on a common consequence (a collider)

r_all = np.corrcoef(talent, beauty)[0, 1]
r_sel = np.corrcoef(talent[hired], beauty[hired])[0, 1]

fig, ax = plt.subplots(figsize=(6.6, 4.4))
ax.scatter(talent[~hired], beauty[~hired], s=8, color="#c9ccd1", alpha=0.5,
           label=f"everyone (r = {r_all:+.2f})")
ax.scatter(talent[hired], beauty[hired], s=16, color="#1f3a5f", alpha=0.8,
           label=f"hired (selected) (r = {r_sel:+.2f})")
# trend line within the selected group
b = np.cov(talent[hired], beauty[hired], bias=True)[0, 1] / np.var(talent[hired])
xs = np.linspace(talent[hired].min(), talent[hired].max(), 30)
ax.plot(xs, beauty[hired].mean() + b * (xs - talent[hired].mean()),
        color="#c0392b", lw=2.2)
ax.set_xlabel("talent"); ax.set_ylabel("beauty")
ax.set_title("Conditioning on a collider invents a correlation")
ax.legend(fontsize=8, frameon=False, loc="upper right")
fig.tight_layout()
plt.show()

Threat 3 — reverse causality

Regression coefficients are symmetric; the data don’t know which way the arrow points. Reverse causality (simultaneity) is when y also causes x. (Wooldridge 2019) More police and more crime appear together — but high-crime cities hire more police, so the naive regression confuses “police cause crime” with “crime causes police.” Likewise income and health: wealth buys health, and health enables earning. When x and y co-determine each other, OLS estimates a tangle of both directions, not the effect you want. (Wooldridge 2019)

Threat 4 — measurement error and attenuation

If your regressor is measured with noise — self-reported income, a proxy for ability, a mismeasured treatment — the coefficient is biased, and in a predictable direction. Under classical measurement error, the bias is attenuation: the coefficient is pulled toward zero. (Wooldridge 2019) The intuition: noise in x adds variation that carries no signal about y, so the estimated slope is diluted — a real effect looks weaker than it is. Mismeasure the outcome and you mostly lose precision; mismeasure the regressor and you bias the effect downward.

The verdict: you need a design, not just controls

Put the threats together and the lesson is stark. Adding control variables cannot, by itself, deliver a credible causal effect — it addresses observed confounders while leaving unobserved confounding, reverse causality, and measurement error untouched, and it can create bias through bad controls. (Angrist and Pischke 2009) Selection-on-observables is an assumption you must argue for, not a box the software ticks.

What does deliver credibility is a research design that makes exogeneity defensible by construction: (Angrist and Pischke 2009)

  • Randomized experiments — random assignment forces X \perp u, the gold standard.
  • Instrumental variables — a lever that moves X but affects Y only through X.
  • Natural experiments — regression discontinuity, difference-in-differences — where nature or policy supplies as-good-as-random variation.

This is the hinge of the whole book. Regression is the engine every one of those designs runs on — but the design is what licenses the causal claim. Part V is the study of building that license.

Common pitfalls

WarningWhere this goes wrong

  • “I controlled for confounders, so it’s causal.” Only for the confounders you observed. Unobserved ones remain, and there’s no test for exogeneity. Controls build credibility, not proof. (Angrist and Pischke 2009)

  • Controlling for everything available. Post-treatment (bad) controls and colliders add bias. Control for predetermined variables; never for outcomes of the treatment. (Angrist and Pischke 2009)

  • Ignoring the direction of causation. If y could plausibly cause x, the coefficient blends both directions. Ask which way the arrows run before interpreting. (Wooldridge 2019)

  • Trusting a precise coefficient on a noisy regressor. Measurement error attenuates toward zero; a small “insignificant” effect may be a real effect, diluted. (Wooldridge 2019)

  • Believing a tiny standard error rescues a biased estimate. Inference quantifies sampling noise, not specification error. A precise estimate of the wrong number is still the wrong number. (Wooldridge 2019)

From the sources

NoteWho says what

Appendix · the mathematics

When OLS is causal. Writing the structural model y=\beta x + u (the data-generating equation whose \beta is the true causal effect, as opposed to a merely descriptive fit), OLS estimates \hat\beta \xrightarrow{p} \beta + \dfrac{\operatorname{Cov}(x,u)}{\operatorname{Var}(x)}, where \xrightarrow{p} denotes the large-sample (probability) limit — the value OLS converges to as n\to\infty. The second term — the bias — vanishes iff \operatorname{Cov}(x,u)=0, i.e. exogeneity. Every threat below is a reason \operatorname{Cov}(x,u)\neq 0.

Attenuation, derived. Let the true regressor be x^* but you observe x=x^*+e with classical error (e independent of x^* and u, variance \sigma_e^2). Then (writing \operatorname{plim}, the probability limit, for that same large-sample value) \operatorname{plim}\hat\beta = \beta\cdot\frac{\sigma_{x^*}^2}{\sigma_{x^*}^2+\sigma_e^2} = \beta\cdot\lambda, \qquad 0<\lambda<1, so the estimate is the truth times a reliability ratio \lambda<1 — shrunk toward zero. The noisier the measure, the smaller \lambda, the worse the attenuation.

Why a collider induces dependence. If C = x + y + \varepsilon is a common consequence and x,y are marginally independent, then conditioning on C=c ties them: y \approx c - x - \varepsilon, so within a stratum of C, higher x forces lower y — a negative conditional correlation from a zero marginal one. Controlling for a post-treatment collider does exactly this.

Simultaneity bias. With y=\beta x+u and x=\gamma y+v, solving the system makes \operatorname{Cov}(x,u)\neq0, so OLS is inconsistent — the coefficient mixes \beta and \gamma. The fix is an instrument that shifts x without touching u — the subject of Part V.

\qquad 0<\lambda<1,
$$
so the estimate is the truth times a **reliability ratio** $\lambda<1$ — shrunk toward zero. The
noisier the measure, the smaller $\lambda$, the worse the attenuation.

**Why a collider induces dependence.** If $C = x + y + \varepsilon$ is a common consequence and $x,y$
are marginally independent, then conditioning on $C=c$ ties them: $y \approx c - x - \varepsilon$, so
*within* a stratum of $C$, higher $x$ forces lower $y$ — a negative conditional correlation from a zero
marginal one. Controlling for a post-treatment collider does exactly this.

**Simultaneity bias.** With $y=\beta x+u$ and $x=\gamma y+v$, solving the system makes
$\operatorname{Cov}(x,u)\neq0$, so OLS is inconsistent — the coefficient mixes $\beta$ and $\gamma$.
The fix is an instrument that shifts $x$ without touching $u$ — the subject of
[Part V](../05-causal-inference/index.qmd).

:::
