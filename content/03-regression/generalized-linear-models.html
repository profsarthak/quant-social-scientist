Generalized Linear Models: One Family

  • Show All Code
  • Hide All Code
  • View Source

OLS, logistic, and Poisson regression aren’t three tools — they’re one. Swap the outcome’s distribution and a link function, and the whole regression machine keeps working.

You’ve now met regression for continuous, binary, and (next) count outcomes. The generalized linear model (GLM) reveals they are the same model in three costumes. Keep the linear predictor X\beta; change two things — the distribution of the outcome and the link function that connects it to X\beta — and OLS becomes logistic becomes Poisson. Learn the family and you stop memorising separate models and start configuring the right one.

Why it matters

OLS assumes a continuous, roughly Normal outcome. Force it onto a 0/1 or a count and it makes “nonsensical predictions” — probabilities above 100%, negative counts. (McNulty 2021) The GLM framework is the principled fix: it keeps everything you know about linear regression — coefficients, a linear predictor, inference — and bends the output into the valid range for whatever outcome you have. It is the umbrella that unifies half of Part III.

The three components

Every GLM is built from exactly three parts: (McNulty 2021)

  1. Random component — the distribution of the outcome Y, drawn from the exponential family (Gaussian, Binomial, Poisson, and more). This is what you change to match the data type.
  2. Systematic component — the linear predictor \eta = X\beta, the familiar linear combination of coefficients and predictors. This never changes.
  3. Link function — a function g(\cdot) that connects the mean of the outcome to the linear predictor: g(\mu) = X\beta. This is the hinge between the two.

The link function does the work

The link is what keeps predictions sensible, and choosing it is choosing the model: (McNulty 2021)

Model Outcome Link g(\mu) Keeps predictions…
OLS continuous identity \mu (any real number)
Logistic binary logit \ln\frac{\mu}{1-\mu} (log-odds) in [0,1]
Poisson count log \ln\mu positive

Why a link is needed: a raw linear predictor X\beta ranges over all of (-\infty,\infty), but a probability must live in [0,1] and a count must be non-negative. The link (and its inverse) constrains the output so it can’t escape its valid range. (McNulty 2021) The logit squashes the line into an S-curve between 0 and 1; the log link’s inverse, \exp(X\beta), keeps counts positive.

Show the figure code 
import numpy as np
import matplotlib.pyplot as plt

eta = np.linspace(-4, 4, 300)          # the linear predictor X*beta

identity = eta
logistic = 1 / (1 + np.exp(-eta))      # inverse logit -> probability in [0,1]
poisson  = np.exp(eta)                 # inverse log   -> positive count

fig, axes = plt.subplots(1, 3, figsize=(9.4, 3.4))
for ax, y, title, col, note in [
    (axes[0], identity, "identity link (OLS)", "#1f3a5f", "any real value"),
    (axes[1], logistic, "logit link (logistic)", "#3f8f6f", "constrained to [0, 1]"),
    (axes[2], poisson,  "log link (Poisson)",   "#9a6a2f", "constrained > 0")]:
    ax.plot(eta, y, color=col, lw=2.6)
    ax.set_title(title, fontsize=10, color=col, fontweight="bold")
    ax.set_xlabel(r"linear predictor  $X\beta$")
    ax.text(0.5, 0.04, note, transform=ax.transAxes, ha="center", fontsize=8.5, style="italic")
axes[0].set_ylabel("predicted mean  $\\mu$")
axes[1].set_ylim(-0.05, 1.05); axes[1].axhline(0, color="#ccc", lw=0.8); axes[1].axhline(1, color="#ccc", lw=0.8)
axes[2].set_ylim(0, 20)
fig.tight_layout()
plt.show()

Estimation: maximum likelihood

Because the link makes the model nonlinear, there’s no closed-form OLS formula. GLMs are fit by maximum likelihood — find the \beta that makes the observed data most probable under the chosen distribution. (McNulty 2021) (Numerically, by iteratively reweighted least squares, but the principle is pure MLE.) This is why logit/probit, Poisson, and Cox survival all share the same inferential machinery — they’re all GLMs (or close cousins) fit by likelihood.

Common pitfalls

WarningWhere this goes wrong

  • Using OLS on a bounded outcome. A linear probability model can predict below 0 or above 1; a line on counts predicts negatives. Pick the link that fits the range. (McNulty 2021)

  • Reading GLM coefficients on the linear-predictor scale. A logit coefficient is in log-odds, a Poisson coefficient in log-counts; back-transform (the inverse link) before interpreting in real units. (McNulty 2021)

  • Forgetting which distribution you assumed. The random component encodes a variance assumption (e.g. Poisson’s mean = variance); if it’s wrong, the SEs are wrong even when the link is right.

  • Treating GLM as a different beast from OLS. It’s the same linear predictor and MLE machinery; OLS is just the identity-link, Gaussian special case.

From the sources

NoteWho says what
  • McNulty, Explanatory Statistical Modeling in R — the GLM as the unifying family over OLS, logistic, and Poisson; the three components (random/exponential-family response, systematic linear predictor \eta=X\beta, link function g(\mu)=X\beta); the standard links (identity/logit/log) and why a link is needed to avoid “nonsensical predictions” (probabilities >1, negative counts); estimation by maximum likelihood (no closed form).
  • Wooldridge, Introductory Econometrics / Moffatt, Experimetrics — the binary and count members of the family and their likelihood-based fitting.

Connections

  • Generalises: simple & multiple regression (identity link) and logit/probit (logit link).
  • Specialises to: count-data models (log link) and underlies survival/Cox.
  • Runs on: maximum likelihood — the estimation engine for every non-Gaussian member.
