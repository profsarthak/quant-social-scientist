Heteroskedasticity, Robust Errors & GLS

  • Show All Code
  • Hide All Code
  • View Source

When the error variance isn’t constant, OLS still hits the right answer on average — but its standard errors lie, and every t-test built on them is wrong. Two cures: be agnostic (robust SEs) or model it (GLS).

The Gauss–Markov ideal assumes constant error variance. Real data rarely oblige — spending varies more among the rich, test scores more in big schools. Under this heteroskedasticity, OLS coefficients stay unbiased, but the usual standard errors are biased, so your t- and F-tests are invalid. The fixes split into two philosophies: robust standard errors that work without knowing the variance pattern, and (feasible) GLS that models it to recover efficiency.

Why it matters

Heteroskedasticity is the rule, not the exception, in cross-sectional social data — and its danger is subtle: nothing looks wrong (the point estimates are fine), but the inference silently misleads, and the problem does not vanish as the sample grows (Wooldridge 2019). Knowing how to detect it and which correction to use is basic hygiene for any regression you’ll defend.

What it is, and what it costs

Heteroskedasticity means the error variance, conditional on the regressors, is not constant: \operatorname{Var}(u\mid X)=\sigma^2(X) depends on X (Wooldridge 2019). The consequences are specific (Wooldridge 2019):

  • Still unbiased and consistent. These properties need only \mathbb{E}[u\mid X]=0, not constant variance — so OLS gets the right answer on average (Wooldridge 2019).
  • No longer BLUE / efficient. OLS loses its minimum-variance crown among linear unbiased estimators (Wooldridge 2019).
  • Invalid inference. The usual SE formulas are biased, so t, F, and confidence intervals are wrong — and stay wrong even in large samples (Wooldridge 2019).

Robust standard errors: the agnostic fix

The everyday response is heteroskedasticity-robust (White / Huber–Eicker) standard errors (Wooldridge 2019). They adjust the SEs to be valid in large samples regardless of the form of the heteroskedasticity — without your having to model \operatorname{Var}(u\mid X) at all (Wooldridge 2019). This is why “robust SEs” is the modern default. One caveat from Wooldridge: in small samples robust SEs can be more misleading than conventional ones, because they carry higher sampling variability themselves (Wooldridge 2019).

Show the code (OLS + White robust SE by hand) 
import numpy as np
import matplotlib.pyplot as plt

rng = np.random.default_rng(0)
beta1_true = 1.0
n, S = 60, 3000

def fit(x, y):
    X = np.column_stack([np.ones(len(x)), x])
    XtX_inv = np.linalg.inv(X.T @ X)
    b = XtX_inv @ X.T @ y
    e = y - X @ b
    # classical SE
    s2 = (e @ e) / (len(x) - 2)
    se_cl = np.sqrt((s2 * XtX_inv)[1, 1])
    # White (HC0) robust SE: sandwich
    meat = X.T @ (e[:, None]**2 * X)
    Vrob = XtX_inv @ meat @ XtX_inv
    se_rob = np.sqrt(Vrob[1, 1])
    return b[1], se_cl, se_rob

cover_cl = cover_rob = 0
for _ in range(S):
    x = rng.uniform(0, 5, n)
    y = 1.0 + beta1_true * x + (0.3 + 0.8 * x) * rng.normal(size=n)   # heteroskedastic
    b1, se_cl, se_rob = fit(x, y)
    cover_cl += abs(b1 - beta1_true) <= 1.96 * se_cl
    cover_rob += abs(b1 - beta1_true) <= 1.96 * se_rob
cover_cl, cover_rob = 100*cover_cl/S, 100*cover_rob/S

fig, (axL, axR) = plt.subplots(1, 2, figsize=(9, 3.8), gridspec_kw={"width_ratios": [1.3, 1]})
xd = rng.uniform(0, 5, n); yd = 1.0 + xd + (0.3 + 0.8*xd)*rng.normal(size=n)
axL.scatter(xd, yd, s=12, color="#9aa0a6", alpha=0.6)
xs = np.linspace(0, 5, 30); axL.plot(xs, 1+xs, color="#1f4e79", lw=2)
axL.set_title("Heteroskedastic data (spread grows with x)"); axL.set_xlabel("x"); axL.set_ylabel("y")

bars = axR.bar(["classical\nOLS SE", "robust\n(White) SE"], [cover_cl, cover_rob],
               color=["#c0392b", "#1f8a4c"])
axR.axhline(95, color="#444", ls="--", lw=1.3); axR.text(1.5, 95.4, "95% target", fontsize=8, ha="right")
axR.set_ylim(80, 100); axR.set_ylabel("actual CI coverage (%)")
for b, v in zip(bars, [cover_cl, cover_rob]):
    axR.text(b.get_x()+b.get_width()/2, v+0.3, f"{v:.0f}%", ha="center", fontsize=9, fontweight="bold")
axR.set_title("Robust SEs restore correct coverage")
fig.tight_layout()
plt.show()

Formal tests

Two standard tests of the null H_0: homoskedasticity (Wooldridge 2019):

  • Breusch–Pagan. Regress the squared OLS residuals on the regressors: \hat u^2 = \delta_0 + \delta_1 x_1 + \cdots + \delta_k x_k + v. The null is \delta_1=\cdots=\delta_k=0 (variance unrelated to X); a significant F/LM rejects it (Wooldridge 2019).
  • White test. More general — regress \hat u^2 on the regressors, their squares, and all cross-products. A parsimonious version uses just the fitted values and their square, \hat u^2 = \delta_0 + \delta_1\hat y + \delta_2\hat y^2 + \text{error} (Wooldridge 2019).

WLS and feasible GLS: model the variance

When you can model the variance, generalized least squares (GLS) restores efficiency (Wooldridge 2019):

  • Weighted least squares (WLS). If \operatorname{Var}(u_i\mid x_i)=\sigma^2 h_i, divide every variable (including the intercept) by \sqrt{h_i} (Wooldridge 2019). This down-weights high-variance observations, making the transformed errors homoskedastic and the estimator BLUE (Wooldridge 2019).
  • Feasible GLS (FGLS). When h_i is unknown, estimate it — typically model the variance as an exponential (to stay positive): regress \log(\hat u^2) on the regressors, exponentiate the fitted values to get \hat h_i = \exp(\hat g_i), and use those as WLS weights (Wooldridge 2019).
TipThe efficiency–robustness trade-off

FGLS is asymptotically more efficient than OLS if the variance model is correct. But if the variance model is wrong, WLS can be less efficient than OLS — so OLS with robust SEs is the “safer,” more interpretable default (Wooldridge 2019). Reach for FGLS when you genuinely know the variance form (or efficiency is critical); otherwise, robust SEs.

The time-series cousin: serial correlation

In time series, errors are often serially correlated — today’s error related to yesterday’s (Wooldridge 2019). The parallels to heteroskedasticity are close: with strictly exogenous regressors OLS stays unbiased/consistent but inefficient, and the usual SEs and tests are invalid (Wooldridge 2019). Detection and cures (Wooldridge 2019; Ashley 2012):

  • Durbin–Watson — the classic AR(1) test; ≈ 2 means no correlation, near 0 means strong positive. Often inconclusive, and invalid with a lagged dependent variable (Wooldridge 2019).
  • Breusch–Godfrey (LM) — more general: regress OLS residuals on the regressors and lagged residuals \hat u_{t-1},\dots,\hat u_{t-q} (Wooldridge 2019).
  • Newey–West (HAC) SEs — robust to both heteroskedasticity and arbitrary serial correlation in large samples (Wooldridge 2019).
  • Cochrane–Orcutt / Prais–Winsten — FGLS via a quasi-differencing transform that removes AR(1), given an estimate of \rho (Wooldridge 2019).
  • Dynamic respecification. Ashley’s emphasis: serial correlation is often a symptom of missing dynamics — adding lagged terms can “cure” it by specifying the model properly (Ashley 2012).

Common pitfalls

WarningWhere this goes wrong
  • Trusting default OLS SEs on cross-sectional data. Assume heteroskedasticity; use robust SEs by default. (Wooldridge 2019)
  • Using robust SEs in a tiny sample. They have high variability themselves and can mislead at small n. (Wooldridge 2019)
  • Applying WLS with a wrong variance model. You can end up less efficient than plain OLS — and biased if the weights are endogenous. (Wooldridge 2019)
  • Treating serial correlation as just a SE problem. It often signals a misspecified dynamic model; fix the specification, don’t only patch the SEs. (Ashley 2012)
  • Reading Durbin–Watson with a lagged dependent variable. It’s invalid there — use Breusch–Godfrey. (Wooldridge 2019)

NoteWho says what

Appendix · the mathematics

The sandwich (White) estimator. OLS gives \hat\beta=(X'X)^{-1}X'y with \operatorname{Var}(\hat\beta\mid X)=(X'X)^{-1}\big(X'\,\Omega\,X\big)(X'X)^{-1}, where \Omega=\operatorname{diag}(\sigma_i^2). Under homoskedasticity \Omega=\sigma^2 I and this collapses to \sigma^2(X'X)^{-1}. White’s insight: you don’t need each \sigma_i^2 — replace the unknown “meat” X'\Omega X=\sum_i \sigma_i^2 x_i x_i' with \sum_i \hat u_i^2\, x_i x_i' (HC0), which is consistent for it. That is exactly the meat = X.T @ (e**2 * X) line in the code; the bread is (X'X)^{-1} on either side.

Why WLS is BLUE. Dividing through by \sqrt{h_i} gives errors u_i/\sqrt{h_i} with variance \sigma^2 h_i/h_i=\sigma^2 — constant. Gauss–Markov now applies to the transformed model, so WLS is BLUE. Equivalently, GLS is \hat\beta_{\text{GLS}}=(X'\Omega^{-1}X)^{-1}X'\Omega^{-1}y, the variance-weighted normal equations.

Newey–West. Generalises the sandwich meat to include lagged cross-products \sum_{\ell} w_\ell \sum_t \hat u_t \hat u_{t-\ell}(x_t x_{t-\ell}' + x_{t-\ell}x_t') with Bartlett weights w_\ell tapering to zero — making the SE robust to serial correlation up to the chosen lag.

Connections

  • Builds on: the Gauss–Markov assumptions (homoskedasticity is the one that fails here), regression diagnostics (detecting it), and inference for regression (the tests it invalidates).

## Formal tests

Two standard tests of the null $H_0$: homoskedasticity [@wooldridge_intro]:

- **Breusch–Pagan.** Regress the **squared OLS residuals** on the regressors:
  $\hat u^2 = \delta_0 + \delta_1 x_1 + \cdots + \delta_k x_k + v$. The null is
  $\delta_1=\cdots=\delta_k=0$ (variance unrelated to $X$); a significant $F$/LM rejects it
  [@wooldridge_intro].
- **White test.** More general — regress $\hat u^2$ on the regressors, **their squares, and all
  cross-products**. A parsimonious version uses just the fitted values and their square,
  $\hat u^2 = \delta_0 + \delta_1\hat y + \delta_2\hat y^2 + \text{error}$ [@wooldridge_intro].

## WLS and feasible GLS: model the variance

When you can *model* the variance, **generalized least squares (GLS)** restores efficiency
[@wooldridge_intro]:

- **Weighted least squares (WLS).** If $\operatorname{Var}(u_i\mid x_i)=\sigma^2 h_i$, **divide every variable
  (including the intercept) by $\sqrt{h_i}$** [@wooldridge_intro]. This **down-weights high-variance
  observations**, making the transformed errors homoskedastic and the estimator **BLUE** [@wooldridge_intro].
- **Feasible GLS (FGLS).** When $h_i$ is unknown, **estimate** it — typically model the variance as an
  exponential (to stay positive): regress $\log(\hat u^2)$ on the regressors, exponentiate the fitted values
  to get $\hat h_i = \exp(\hat g_i)$, and use those as WLS weights [@wooldridge_intro].

::: {.callout-tip title="The efficiency–robustness trade-off"}
FGLS is **asymptotically more efficient than OLS *if* the variance model is correct**. But if the variance
model is **wrong**, WLS can be **less** efficient than OLS — so **OLS with robust SEs is the "safer,"
more interpretable default** [@wooldridge_intro]. Reach for FGLS when you genuinely know the variance form
(or efficiency is critical); otherwise, robust SEs.
:::

## The time-series cousin: serial correlation

In time series, errors are often **serially correlated** — today's error related to yesterday's
[@wooldridge_intro]. The parallels to heteroskedasticity are close: with strictly exogenous regressors OLS
stays **unbiased/consistent but inefficient**, and the **usual SEs and tests are invalid**
[@wooldridge_intro]. Detection and cures [@wooldridge_intro; @ashley_econometrics]:

- **Durbin–Watson** — the classic AR(1) test; ≈ 2 means no correlation, near 0 means strong positive. Often
  inconclusive, and invalid with a lagged dependent variable [@wooldridge_intro].
- **Breusch–Godfrey (LM)** — more general: regress OLS residuals on the regressors **and lagged residuals**
  $\hat u_{t-1},\dots,\hat u_{t-q}$ [@wooldridge_intro].
- **Newey–West (HAC)** SEs — robust to **both** heteroskedasticity and **arbitrary serial correlation** in
  large samples [@wooldridge_intro].
- **Cochrane–Orcutt / Prais–Winsten** — FGLS via a **quasi-differencing** transform that removes AR(1),
  given an estimate of $\rho$ [@wooldridge_intro].
- **Dynamic respecification.** Ashley's emphasis: serial correlation is often a **symptom of missing
  dynamics** — adding lagged terms can **"cure"** it by specifying the model properly
  [@ashley_econometrics].

## Common pitfalls

::: {.callout-warning title="Where this goes wrong"}
- **Trusting default OLS SEs on cross-sectional data.** Assume heteroskedasticity; use robust SEs by default.
  [@wooldridge_intro]
- **Using robust SEs in a tiny sample.** They have high variability themselves and can mislead at small $n$.
  [@wooldridge_intro]
- **Applying WLS with a wrong variance model.** You can end up *less* efficient than plain OLS — and biased
  if the weights are endogenous. [@wooldridge_intro]
- **Treating serial correlation as just a SE problem.** It often signals a misspecified dynamic model; fix
  the specification, don't only patch the SEs. [@ashley_econometrics]
- **Reading Durbin–Watson with a lagged dependent variable.** It's invalid there — use Breusch–Godfrey.
  [@wooldridge_intro]
:::

## Appendix · the mathematics {.unnumbered}

::: {.math-appendix}

**The sandwich (White) estimator.** OLS gives $\hat\beta=(X'X)^{-1}X'y$ with
$\operatorname{Var}(\hat\beta\mid X)=(X'X)^{-1}\big(X'\,\Omega\,X\big)(X'X)^{-1}$, where
$\Omega=\operatorname{diag}(\sigma_i^2)$. Under homoskedasticity $\Omega=\sigma^2 I$ and this collapses to
$\sigma^2(X'X)^{-1}$. White's insight: you don't need each $\sigma_i^2$ — replace the unknown "meat"
$X'\Omega X=\sum_i \sigma_i^2 x_i x_i'$ with $\sum_i \hat u_i^2\, x_i x_i'$ (HC0), which is consistent for it.
That is exactly the `meat = X.T @ (e**2 * X)` line in the code; the **bread** is $(X'X)^{-1}$ on either side.

**Why WLS is BLUE.** Dividing through by $\sqrt{h_i}$ gives errors $u_i/\sqrt{h_i}$ with variance
$\sigma^2 h_i/h_i=\sigma^2$ — constant. Gauss–Markov now applies to the transformed model, so WLS is BLUE.
Equivalently, GLS is $\hat\beta_{\text{GLS}}=(X'\Omega^{-1}X)^{-1}X'\Omega^{-1}y$, the variance-weighted
normal equations.

**Newey–West.** Generalises the sandwich meat to include lagged cross-products
$\sum_{\ell} w_\ell \sum_t \hat u_t \hat u_{t-\ell}(x_t x_{t-\ell}' + x_{t-\ell}x_t')$ with Bartlett weights
$w_\ell$ tapering to zero — making the SE robust to serial correlation up to the chosen lag.

:::
