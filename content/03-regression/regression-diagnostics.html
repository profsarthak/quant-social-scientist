Diagnostics & Robust Inference

  • Show All Code
  • Hide All Code
  • View Source

Check the assumptions, then repair the standard errors — because real data rarely reads the textbook.

The Gauss–Markov assumptions are rarely exactly true. The good news: the most common violations — non-constant error variance, clustered data, correlated regressors — usually leave the coefficients unbiased and damage only the standard errors. So diagnostics is mostly about detecting those violations and repairing the inference, not throwing out the model. This entry is the practitioner’s checklist.

Why it matters

A regression can be unbiased and still lie to you — not through its coefficients, but through its standard errors. (Wooldridge 2019) If the SEs are wrong, every t-test, p-value, and confidence interval built on them is wrong, and you’ll claim significance you haven’t earned (or miss effects that are real). The reassuring theme of this entry: heteroskedasticity, clustering, and multicollinearity do not bias OLS — they distort precision — so the fixes are about honest uncertainty, and they’re cheap. Knowing which problems threaten the coefficients (functional-form misspecification, influential outliers) versus only the standard errors is the core diagnostic skill.

Heteroskedasticity

Heteroskedasticity is non-constant error variance: \operatorname{Var}(u\mid x) changes with the regressors, violating MLR.5. (Wooldridge 2019) It is the rule, not the exception, in social data — spending varies more among rich households than poor ones, test scores more in big schools than small.

Consequences. OLS stays unbiased and consistent — the coefficients are fine. But OLS is no longer BLUE, and the usual standard-error formulas are invalid, so t-tests and intervals are unreliable even in large samples. (Wooldridge 2019)

Detection. (Wooldridge 2019; Ashley 2012)

  • Graphically: plot residuals against fitted values and look for a fan or cone — scatter that widens (or narrows) as the fitted value grows.
  • Breusch–Pagan test: regress the squared residuals \hat u^2 on the regressors; a significant F/LM statistic rejects homoskedasticity.
  • White test: the same idea with squares and cross-products added (a popular shortcut regresses \hat u^2 on \hat y and \hat y^2).
Show the figure code 
import numpy as np
import matplotlib.pyplot as plt

rng = np.random.default_rng(6)
n = 300
x = rng.uniform(0, 10, n)
fitted = 2 + 1.2 * x

resid_homo = rng.normal(0, 1.5, n)                 # constant variance
resid_hetero = rng.normal(0, 0.3 + 0.45 * x)       # variance grows with x (fan)

fig, (axL, axR) = plt.subplots(1, 2, figsize=(9.2, 3.6), sharey=True)
for ax, resid, title in [(axL, resid_homo, "Homoskedastic: even band"),
                         (axR, resid_hetero, "Heteroskedastic: a widening fan")]:
    ax.scatter(fitted, resid, s=12, color="#1f3a5f", alpha=0.55)
    ax.axhline(0, color="#c0392b", lw=1.3)
    ax.set_title(title, fontsize=10)
    ax.set_xlabel("fitted value  ŷ")
axL.set_ylabel("residual  û")
fig.suptitle("Residuals vs fitted — the first plot to draw", fontsize=12)
fig.tight_layout()
plt.show()

The fix. Use heteroskedasticity-robust standard errors (White / Huber–White / “robust” SEs), valid in large samples whatever the form of the heteroskedasticity. (Wooldridge 2019) They cost nothing and change only the SEs, not the coefficients — which is why they are now the default in much professional research: when in doubt, report robust SEs. (Wooldridge 2019) (Ashley’s example: a regression of state death rates on marriage rates shows far wider scatter for populous states — severe heteroskedasticity that invalidates the naive SEs. (Ashley 2012))

Weighted and generalized least squares

Robust SEs correct the inference but leave OLS no longer efficient — under heteroskedasticity OLS is unbiased but not BLUE. The classical alternative restores efficiency: if you know (or can model) how the error variance changes, weighted least squares (WLS) down-weights the noisy, high-variance observations and up-weights the precise ones. (Wooldridge 2019) WLS minimises a weighted sum of squared residuals, with each observation weighted by the inverse of its error variance 1/\sigma_i^2 — so a point with twice the variance counts half as much. It is the special case of generalized least squares (GLS) for a diagonal (uncorrelated-but-unequal) error structure, and under the assumed variance form it is BLUE. In practice \sigma_i^2 is unknown and must be estimated first (model the variance, then weight), giving feasible GLS (FGLS). (Wooldridge 2019)

The modern trade-off: WLS/GLS is more efficient if you get the variance model right, but inconsistent for the SEs if you get it wrong; robust SEs on plain OLS are less efficient but safe. Today most applied researchers prefer OLS-with-robust-SEs for its robustness, reserving WLS/GLS for cases where the variance structure is genuinely known (e.g., grouped/averaged data, where the variance scales with group size). (Wooldridge 2019)

Clustered standard errors

A cousin of heteroskedasticity. When observations fall into clusters — students within classrooms, repeated measurements on the same person, households within villages — the errors are correlated within each cluster because they share unobserved influences (the same teacher, the same person). (Ashley 2012) Ordinary SEs assume independence, so they come out too small, producing over-optimistic t-statistics and false significance. (Ashley 2012)

The damage is quantified by the Moulton factor, 1 + (n-1)\rho_e, where \rho_e is the intra-cluster correlation and n the cluster size. (Ashley 2012) The fix is cluster-robust standard errors, which allow arbitrary correlation within clusters while assuming independence across them.

NoteHow big can the clustering inflation be? (STAR)

In the Tennessee STAR class-size experiment, test scores were correlated within classrooms at \rho_e = 0.31. With realistic class sizes the Moulton factor was about 7 — meaning the honest standard errors were roughly \sqrt{7} \approx \mathbf{2.65} times the naive ones. Ignore clustering and you’d overstate your precision by a factor of nearly three. (Ashley 2012)

Multicollinearity

Multicollinearity is high correlation among regressors. (Wooldridge 2019) Its symptoms are distinctive: large standard errors, coefficients that swing wildly with small data changes, and the tell-tale combination of a high R^2 with few individually significant t-statistics (the model explains a lot collectively, but can’t credit any single variable). (Wooldridge 2019)

Detect it with the variance inflation factor, \text{VIF}_j = 1/(1 - R_j^2), where R_j^2 comes from regressing x_j on the other regressors; a common rule of thumb flags \text{VIF} > 10. (Wooldridge 2019) This is exactly the (1-R_j^2) term in the coefficient-variance formula — multicollinearity is variance inflation, literally.

TipWhy it usually isn’t a crisis

Multicollinearity is a data problem, not an assumption violation — Arthur Goldberger mockingly called it “micronumerosity” (not enough independent information), the same nuisance as a small sample. (Wooldridge 2019) OLS stays unbiased; you just can’t separate the tangled variables precisely. The practical verdict: if your coefficient of interest is still significant despite the collinearity, you don’t need to worry — and “dropping a collinear control” can reintroduce omitted-variable bias, which is worse. (Wooldridge 2019)

Outliers, leverage, and influence

OLS minimises squared residuals, so it is acutely sensitive to extreme points. (Ashley 2012) Three distinct ideas: (Ashley 2012)

  • Outlier: an observation whose actual y is far from its fitted \hat y (large studentized residual).
  • Leverage: an observation with an extreme value of x — it has the potential to swing the slope, like a long lever arm.
  • Influence: an observation whose removal materially changes the estimates. Cook’s distance combines residual and leverage into one influence score.

A high-leverage outlier is the dangerous combination — one point can tilt the whole line.

NoteNevada and the bridal chapels

In 1980 Census data, Nevada is a wild outlier on marriage rates (its wedding-chapel industry). In a death-on-marriage regression, including Nevada pulled the slope from 0.89 down to 0.82 — a single state moving the headline coefficient. (Ashley 2012) Always check whether one point is driving your result.

Functional-form misspecification

If you fit a line to a relationship that’s really a curve — omitting a needed log or quadratic — the estimates are biased and inconsistent. (Wooldridge 2019) The Ramsey RESET test is a general check: (Wooldridge 2019)

  1. Estimate your model and get the fitted values \hat y.
  2. Re-estimate with powers of the fitted values (\hat y^2, \hat y^3) added as regressors.
  3. F-test their joint significance. If significant, the original specification is rejected — you’ve missed a nonlinearity.

A caveat Wooldridge stresses: RESET tests functional form only, not omitted variables that happen to be uncorrelated with the included regressors. (Wooldridge 2019)

Common pitfalls

WarningWhere this goes wrong

  • Treating heteroskedasticity/clustering as bias. They corrupt standard errors, not coefficients. The fix is robust/clustered SEs, not a different estimator. (Wooldridge 2019)

  • Failing to cluster grouped data. Independent-looking SEs on clustered data can overstate precision several-fold (STAR’s ×2.65). Cluster at the level of treatment assignment. (Ashley 2012)

  • Panicking about multicollinearity. If your key coefficient is significant, collinearity among controls is harmless. Don’t drop a needed control just to lower a VIF — that risks OVB. (Wooldridge 2019)

  • Ignoring influential points — or deleting them reflexively. Check leverage and Cook’s distance; investigate an influential point (is it an error? a real but special case?) before keeping or dropping it. (Ashley 2012)

  • Skipping the residual plot. The single residuals-vs-fitted plot reveals heteroskedasticity and functional-form problems in seconds. Draw it every time.

Appendix · the mathematics

Robust (White) variance. OLS gives \hat\beta=\beta+(X'X)^{-1}X'u, so \operatorname{Var}(\hat\beta\mid X)=(X'X)^{-1}\big(X'\,\Omega\,X\big)(X'X)^{-1} with \Omega=\operatorname{diag}(\sigma_i^2). Under homoskedasticity \Omega=\sigma^2 I and this collapses to the usual \sigma^2(X'X)^{-1}. The White estimator replaces the unknown \sigma_i^2 with the squared residuals \hat u_i^2: \widehat{\operatorname{Var}}=(X'X)^{-1}\big(\sum_i \hat u_i^2 x_i x_i'\big)(X'X)^{-1} — consistent for any heteroskedasticity, no model of it required.

Cluster-robust variance. Replace the middle “meat” with a sum over clusters g: \sum_g X_g'\,\hat u_g \hat u_g'\,X_g, allowing arbitrary within-cluster correlation. The Moulton factor 1+(n-1)\rho_e is the ratio of the correct variance to the naive one in the simple equicorrelated case — at \rho_e=0.31, n\approx 20, it is \approx 7.

VIF and the variance formula. From \operatorname{Var}(\hat\beta_j)=\sigma^2/[\text{SST}_j(1-R_j^2)], the factor 1/(1-R_j^2)=\text{VIF}_j is exactly how much the variance is inflated relative to an orthogonal regressor. R_j^2=0.9\Rightarrow\text{VIF}=10\Rightarrow SE inflated by \sqrt{10}\approx3.2.

called it **"micronumerosity"** (not enough independent information), the same nuisance as a small
sample. [@wooldridge_intro] OLS stays **unbiased**; you just can't separate the tangled variables
precisely. The practical verdict: **if your coefficient of interest is still significant despite the
collinearity, you don't need to worry** — and "dropping a collinear control" can reintroduce
[omitted-variable bias](multiple-regression.qmd), which is worse. [@wooldridge_intro]
:::

## Outliers, leverage, and influence

OLS minimises *squared* residuals, so it is acutely sensitive to extreme points. [@ashley_econometrics]
Three distinct ideas: [@ashley_econometrics]

- **Outlier:** an observation whose actual $y$ is far from its fitted $\hat y$ (large **studentized
  residual**).
- **Leverage:** an observation with an extreme value of $x$ — it has the *potential* to swing the
  slope, like a long lever arm.
- **Influence:** an observation whose *removal* materially changes the estimates. **Cook's distance**
  combines residual and leverage into one influence score.

A high-leverage outlier is the dangerous combination — one point can tilt the whole line.

::: {.callout-note title="Nevada and the bridal chapels"}
In 1980 Census data, Nevada is a wild outlier on marriage rates (its wedding-chapel industry). In a
death-on-marriage regression, *including Nevada pulled the slope from 0.89 down to 0.82* — a single
state moving the headline coefficient. [@ashley_econometrics] Always check whether one point is
driving your result.
:::

## Functional-form misspecification

If you fit a line to a relationship that's really a curve — omitting a needed
[log or quadratic](functional-form.qmd) — the estimates are biased and inconsistent.
[@wooldridge_intro] The **Ramsey RESET test** is a general check: [@wooldridge_intro]

1. Estimate your model and get the fitted values $\hat y$.
2. Re-estimate with powers of the fitted values ($\hat y^2$, $\hat y^3$) added as regressors.
3. **F-test** their joint significance. If significant, the original specification is **rejected** —
   you've missed a nonlinearity.

A caveat Wooldridge stresses: RESET tests *functional form only*, not omitted variables that happen
to be uncorrelated with the included regressors. [@wooldridge_intro]

## Common pitfalls

::: {.callout-warning title="Where this goes wrong"}

- **Treating heteroskedasticity/clustering as bias.** They corrupt *standard errors*, not
  coefficients. The fix is robust/clustered SEs, not a different estimator. [@wooldridge_intro]

- **Failing to cluster grouped data.** Independent-looking SEs on clustered data can overstate
  precision several-fold (STAR's ×2.65). Cluster at the level of treatment assignment.
  [@ashley_econometrics]

- **Panicking about multicollinearity.** If your key coefficient is significant, collinearity among
  *controls* is harmless. Don't drop a needed control just to lower a VIF — that risks OVB.
  [@wooldridge_intro]

- **Ignoring influential points — or deleting them reflexively.** Check leverage and Cook's distance;
  *investigate* an influential point (is it an error? a real but special case?) before keeping or
  dropping it. [@ashley_econometrics]

- **Skipping the residual plot.** The single residuals-vs-fitted plot reveals heteroskedasticity *and*
  functional-form problems in seconds. Draw it every time.

:::

## Appendix · the mathematics {.unnumbered}

::: {.math-appendix}

**Robust (White) variance.** OLS gives $\hat\beta=\beta+(X'X)^{-1}X'u$, so
$\operatorname{Var}(\hat\beta\mid X)=(X'X)^{-1}\big(X'\,\Omega\,X\big)(X'X)^{-1}$ with
$\Omega=\operatorname{diag}(\sigma_i^2)$. Under homoskedasticity $\Omega=\sigma^2 I$ and this collapses
to the usual $\sigma^2(X'X)^{-1}$. The **White estimator** replaces the unknown $\sigma_i^2$ with the
squared residuals $\hat u_i^2$: $\widehat{\operatorname{Var}}=(X'X)^{-1}\big(\sum_i \hat u_i^2 x_i
x_i'\big)(X'X)^{-1}$ — consistent for *any* heteroskedasticity, no model of it required.

**Cluster-robust variance.** Replace the middle "meat" with a sum over clusters $g$:
$\sum_g X_g'\,\hat u_g \hat u_g'\,X_g$, allowing arbitrary within-cluster correlation. The Moulton
factor $1+(n-1)\rho_e$ is the ratio of the correct variance to the naive one in the simple equicorrelated
case — at $\rho_e=0.31$, $n\approx 20$, it is $\approx 7$.

**VIF and the variance formula.** From $\operatorname{Var}(\hat\beta_j)=\sigma^2/[\text{SST}_j(1-R_j^2)]$,
the factor $1/(1-R_j^2)=\text{VIF}_j$ is exactly how much the variance is inflated *relative to* an
orthogonal regressor. $R_j^2=0.9\Rightarrow\text{VIF}=10\Rightarrow$ SE inflated by $\sqrt{10}\approx3.2$.

:::
