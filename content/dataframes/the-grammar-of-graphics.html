The Grammar of Graphics

  • Show All Code
  • Hide All Code
  • View Source

A plot is not a chart type you pick from a menu — it’s a mapping from data columns to visual properties, built in layers.

Most people think of plots as a menu — bar chart, scatter, line — and pick one. R for Data Science teaches something deeper and more generative: every plot is a mapping from variables in a tidy table to visual properties (position, colour, size, shape), assembled from interchangeable layers. Learn the grammar and you stop memorising chart types and start composing exactly the picture your question needs.

Why it matters

Visualisation is the step that can surprise you — the one tool that shows you what you didn’t expect. (Wickham et al. 2023) But a menu of fixed chart types is a cage: it can only draw the pictures someone pre-built. ggplot2’s grammar of graphics replaces the menu with a small set of composable pieces, so the space of plots you can build is effectively unlimited — and, more importantly, each plot becomes a precise statement about which variable controls which visual channel. (Wickham et al. 2023)

A plot is a mapping

The core mental shift: a plot is a mapping of variables to aesthetics. An aesthetic is any visual property of the marks on the page — their x-position, y-position, colour, size, shape, transparency. You build a graph by saying which column drives which aesthetic. (Wickham et al. 2023)

library(tidyverse)

ggplot(data = mpg) +
  geom_point(mapping = aes(x = displ, y = hwy, color = class))

Read it as a sentence: take mpg, add a layer of points, and map engine size to the x-axis, highway mileage to the y-axis, and vehicle class to colour. Move class from color to size and the same data tells a different visual story — the mapping is the choice, not the chart type. (Wickham et al. 2023)

Show the figure code 
import numpy as np
import matplotlib.pyplot as plt

rng = np.random.default_rng(7)
n = 60
x = rng.uniform(1.5, 6.5, n)
grp = rng.integers(0, 3, n)                 # a categorical third variable
y = 35 - 3.0 * x + grp * 2.5 + rng.normal(0, 1.5, n)

palette = np.array(["#1f3a5f", "#3f8f6f", "#9a6a2f"])
fig, (axL, axR) = plt.subplots(1, 2, figsize=(8.8, 3.6))

# map group -> COLOR
axL.scatter(x, y, c=palette[grp], s=34, alpha=0.85)
axL.set_title("group → color", fontsize=10.5, fontweight="bold", color="#1f3a5f")
axL.set_xlabel("engine size"); axL.set_ylabel("mileage")

# map group -> SIZE
axR.scatter(x, y, s=(grp + 1) * 45, facecolor="none", edgecolor="#1f3a5f", lw=1.4, alpha=0.85)
axR.set_title("group → size", fontsize=10.5, fontweight="bold", color="#9a6a2f")
axR.set_xlabel("engine size"); axR.set_ylabel("mileage")

fig.tight_layout()
plt.show()

Once you map a variable to an aesthetic, ggplot2 handles scaling for you — it “automatically assigns a unique level of the aesthetic (e.g. a specific shade of blue) to each unique value of the variable,” and draws the legend. (Wickham et al. 2023) You declare the mapping; the system works out the colours, sizes, and key.

Layers all the way down

The grammar’s reach comes from its claim that the pieces are universal. The authors state it precisely: “you can uniquely describe any plot as a combination of a dataset, a geom, a set of mappings, a stat, a position adjustment, a coordinate system, and a faceting scheme.” (Wickham et al. 2023)

  • geom — the kind of mark (points, lines, bars, boxplots).
  • mappings — which variable drives which aesthetic.
  • stat — a transformation applied before drawing (a histogram’s geom is a bar, but its stat bins and counts).
  • position, coordinate system, facets — how marks are arranged, the space they live in, and how the plot is split into small multiples by a variable.

You add layers with +, the visual cousin of the pipe: each layer composes onto the last, so a complex figure is built up from simple, legible parts rather than configured through one monstrous function call. (Wickham et al. 2023)

Common pitfalls

WarningWhere this goes wrong

  • Setting an aesthetic when you meant to map it. color = "blue" inside aes() doesn’t paint the points blue — it creates a constant variable called “blue” and maps it, producing a red legend titled “blue.” To set a constant, put it outside aes(). Mapping vs setting is the classic confusion. (Wickham et al. 2023)

  • Mapping a continuous variable to a discrete aesthetic (e.g. shape), which ggplot can’t sensibly do — match the variable’s type to the aesthetic’s. (Wickham et al. 2023)

  • Overloading aesthetics. Driving colour and size and shape from three different variables at once usually buries the signal. Each aesthetic is a channel; too many channels is noise. (Wickham et al. 2023)

  • Reaching for a chart type instead of a mapping. Ask “which variable should control position? colour?” — not “should this be a bar or a line?” The mapping is the real decision. (Wickham et al. 2023)

From the sources

NoteWho says what
  • Wickham & Grolemund, R for Data Science — ggplot2 as the grammar of graphics; a plot as a mapping of variables to aesthetics (position, colour, size, shape) via aes(); scaling as the automatic assignment of a unique aesthetic level to each value, plus the legend; and the claim that any plot is “a combination of a dataset, a geom, a set of mappings, a stat, a position adjustment, a coordinate system, and a faceting scheme,” composed in layers with +.

## Common pitfalls

::: {.callout-warning title="Where this goes wrong"}

- **Setting an aesthetic when you meant to map it.** `color = "blue"` *inside* `aes()` doesn't paint the
  points blue — it creates a constant variable called "blue" and maps it, producing a red legend titled
  "blue." To *set* a constant, put it *outside* `aes()`. Mapping vs setting is the classic confusion.
  [@wickham_r4ds]

- **Mapping a continuous variable to a discrete aesthetic** (e.g. shape), which `ggplot` can't sensibly do —
  match the variable's type to the aesthetic's. [@wickham_r4ds]

- **Overloading aesthetics.** Driving colour *and* size *and* shape from three different variables at once
  usually buries the signal. Each aesthetic is a channel; too many channels is noise. [@wickham_r4ds]

- **Reaching for a chart type instead of a mapping.** Ask "which variable should control position? colour?"
  — not "should this be a bar or a line?" The mapping is the real decision. [@wickham_r4ds]

:::
