Community Detection & Homophily

  • Show All Code
  • Hide All Code
  • View Source

Between the single node and the whole network lies mesoscale structure — the social circles, factions, and echo chambers. Modularity scores a grouping; Louvain finds a good one.

Networks aren’t uniform — they clump. A community is a set of nodes more densely connected to each other than to the rest, and finding communities reveals the social circles, political factions, topic clusters, and echo chambers that organise a system (Menczer et al. 2020). The standard tool is modularity (a score for any proposed grouping) maximised by the Louvain algorithm — and the clumping itself is usually driven by homophily, the pull of like toward like.

Why it matters

Community structure is where networks meet the social questions this manual cares about: polarisation, segregation, filter bubbles, in-group/out-group dynamics. Detecting it lets you classify nodes by role — core members versus boundary/bridge nodes — and turns “is this network polarised?” into a measurable quantity.

What a community is

A community (module, cluster) is a group of nodes connected more densely internally than to the outside (Menczer et al. 2020). Socially they are friendship circles, liberal-vs-conservative blog blocs, web topic clusters, and echo chambers of like-minded exposure (Menczer et al. 2020). Knowing the partition lets you separate core members from boundary nodes that broker between groups (Menczer et al. 2020).

Modularity: scoring a partition

Modularity Q measures how much denser the within-community links are than expected by chance (Menczer et al. 2020). “By chance” means a configuration model — a randomised network preserving each node’s degree but rewiring at random (Menczer et al. 2020). For an undirected network:

Q = \frac{1}{2m}\sum_{i,j}\Big[A_{ij} - \frac{k_i k_j}{2m}\Big]\,\delta(c_i, c_j),

with m edges, degrees k_i, and \delta(c_i,c_j)=1 when i,j share a community (Menczer et al. 2020). The bracket is observed minus expected internal connection; a high Q means internal links far exceed random fluctuation — a good partition (Menczer et al. 2020).

Louvain: finding the partition

Maximising Q over all partitions is intractable, so the Louvain method greedily approximates it — and is parameter-free and scales to millions of nodes (Menczer et al. 2020):

  1. Local optimisation. Start every node in its own community; repeatedly move each node into the neighbouring community that most increases Q (\Delta Q) until no move helps (Menczer et al. 2020).
  2. Aggregation. Collapse each community into a super-node, building a smaller meta-network whose edge weights sum the underlying links (Menczer et al. 2020).
  3. Recurse the two phases on the meta-network until Q can’t improve (Menczer et al. 2020).
Show the figure code (modularity + Louvain local-moving, by hand) 
import numpy as np
import matplotlib.pyplot as plt

rng = np.random.default_rng(1)
sizes = [13, 13, 13]; N = sum(sizes)
label_true = np.concatenate([[g]*s for g, s in enumerate(sizes)])
A = np.zeros((N, N), int)
for i in range(N):
    for j in range(i+1, N):
        p = 0.45 if label_true[i] == label_true[j] else 0.02
        if rng.random() < p: A[i, j] = A[j, i] = 1
k = A.sum(1); m = A.sum() / 2; adj = [np.where(A[i])[0] for i in range(N)]

def modularity(comm):
    Q = 0.0
    for i in range(N):
        for j in range(N):
            if comm[i] == comm[j]:
                Q += A[i, j] - k[i]*k[j]/(2*m)
    return Q/(2*m)

# Louvain phase 1: local moving
comm = np.arange(N)
improved = True
while improved:
    improved = False
    for i in rng.permutation(N):
        nbr_comms = set(comm[adj[i]]) | {comm[i]}
        best, bestgain = comm[i], 0.0
        for c in nbr_comms:
            kin = sum(1 for nb in adj[i] if comm[nb] == c)          # i's edges into c
            sigtot = k[(comm == c)].sum() - (k[i] if comm[i] == c else 0)
            gain = kin - k[i]*sigtot/(2*m)                          # ∝ ΔQ
            if gain > bestgain + 1e-9: best, bestgain = c, gain
        if best != comm[i]: comm[i] = best; improved = True

# relabel communities 0..K-1
_, comm = np.unique(comm, return_inverse=True)
Q_det = modularity(comm)
Q_rand = modularity(rng.integers(0, comm.max()+1, N))

# layout: clusters apart, jitter within
ctr = {c: (np.cos(2*np.pi*c/(comm.max()+1)), np.sin(2*np.pi*c/(comm.max()+1))) for c in range(comm.max()+1)}
pos = np.array([ctr[comm[i]] for i in range(N)]) + rng.normal(0, 0.18, (N, 2))

fig, ax = plt.subplots(figsize=(7.0, 4.6))
for i in range(N):
    for j in adj[i]:
        if j > i: ax.plot(*zip(pos[i], pos[j]), color="#dcdccf", lw=0.5, zorder=1)
# Communities are NOMINAL -- no community outranks another -- so this uses the
# brand's categorical register (--cat-1..6), which sits at one value/chroma level
# so hue reads as *which* community rather than as rank. Marker shape repeats the
# distinction, because several of these hues converge under red-green colour
# blindness. Indexing the palette directly (rather than c=comm with a colormap)
# also avoids the colormap normalising K communities across a longer list and
# handing two clusters the same colour.
fills   = ["#e8531d", "#2b7a74", "#b0851d", "#3f5c78", "#6f7a37", "#7c4a63"]
markers = ["o", "s", "^", "D", "v", "P"]
for c in range(comm.max() + 1):
    m = comm == c
    ax.scatter(pos[m,0], pos[m,1], color=fills[c % len(fills)], marker=markers[c % len(markers)],
               s=90, zorder=2, edgecolor="#161613", linewidth=1.0)
ax.set_title(f"Louvain recovers the communities:  Q = {Q_det:.2f}  (random partition Q = {Q_rand:.2f})")
ax.axis("off")
fig.tight_layout()
plt.show()

Homophily and assortativity

Why do communities form? Largely homophily — the tendency to connect with similar others (“birds of a feather”) (Menczer et al. 2020). The technical measure is assortativity: the correlation between the attributes of connected nodes (Barabási 2016):

  • Assortative mixing — similar connects to similar (hubs to hubs). Most human social networks are assortative (Barabási 2016).
  • Disassortative mixing — dissimilar connects (hubs to peripheral nodes), common in biological and technological networks (Barabási 2016).

The social consequence is direct: homophily and assortative mixing naturally produce segregated clusters, and with them polarisation and filter bubbles (Menczer et al. 2020) — the network-structural root of the Schelling-style segregation the Dynamics cluster will simulate.

Cautions

WarningCommunity detection is genuinely hard
  • Resolution limit. Modularity optimisation can’t see small communities — it merges any whose total links fall below \sqrt{2m} into larger ones (Menczer et al. 2020).
  • Ill-posed. There is no unique definition of a community, so different algorithms give different answers (Menczer et al. 2020).
  • Degeneracy. Large networks have many distinct partitions with near-optimal Q, so a single “true” community structure may not exist (Menczer et al. 2020).

Common pitfalls

WarningWhere this goes wrong
  • Reifying the partition. Treat the output as a useful grouping, not the truth — run multiple algorithms/seeds and compare. (Menczer et al. 2020)
  • Ignoring the resolution limit. If you need fine structure, use multi-resolution methods, not vanilla modularity. (Menczer et al. 2020)
  • Reading communities as causal groups. Homophily (selection) and influence both produce clusters; the partition alone can’t separate them. (Menczer et al. 2020)
  • Comparing Q across networks. Q depends on size and density; don’t rank different networks by raw modularity. (Barabási 2016)

From the sources

NoteWho says what
  • Menczer, Fortunato & Davis, A First Course in Network Science — communities as denser-within groups (factions, echo chambers); modularity Q=\frac{1}{2m}\sum_{ij}[A_{ij}-k_ik_j/2m]\delta(c_i,c_j) vs the configuration null; the Louvain local-moving + aggregation algorithm; homophily; and the resolution limit, ill-posedness, and degeneracy cautions.
  • Barabási, Network Science — assortativity (assortative vs disassortative mixing) and the size/density dependence of modularity.

Appendix · the mathematics

Why k_ik_j/2m is “expected.” In the configuration model, a stub of node i connects to a stub of node j with probability \propto k_j; summing over i’s k_i stubs gives an expected k_ik_j/2m edges between i and j. Modularity sums (actual - expected) over within-community pairs, so Q>0 means communities have more internal edges than a degree-matched random graph.

The local-moving gain (the code). Moving isolated node i into community C changes modularity by \Delta Q \propto k_{i,C} - \frac{k_i\,\Sigma_{\text{tot}}(C)}{2m}, where k_{i,C} is i’s edge count into C and \Sigma_{\text{tot}}(C) the total degree of C. Greedily picking the C that maximises this is one Louvain sweep — exactly the loop above. Aggregation then repeats it on the community graph, giving Louvain’s near-linear scaling.

Resolution limit, sketched. Because the null term scales with the whole network’s 2m, two small but clearly separate modules can look like noise relative to the global expectation and get merged once their internal links drop below \sim\sqrt{2m} — a global yardstick applied to local structure.

  modularity. [@barabasi_network]
:::

## Appendix · the mathematics {.unnumbered}

::: {.math-appendix}

**Why $k_ik_j/2m$ is "expected."** In the configuration model, a stub of node $i$ connects to a stub of node
$j$ with probability $\propto k_j$; summing over $i$'s $k_i$ stubs gives an expected $k_ik_j/2m$ edges between
$i$ and $j$. Modularity sums (actual $-$ expected) over **within-community** pairs, so $Q>0$ means
communities have more internal edges than a degree-matched random graph.

**The local-moving gain (the code).** Moving isolated node $i$ into community $C$ changes modularity by
$\Delta Q \propto k_{i,C} - \frac{k_i\,\Sigma_{\text{tot}}(C)}{2m}$, where $k_{i,C}$ is $i$'s edge count into
$C$ and $\Sigma_{\text{tot}}(C)$ the total degree of $C$. Greedily picking the $C$ that maximises this is one
Louvain sweep — exactly the loop above. Aggregation then repeats it on the community graph, giving Louvain's
near-linear scaling.

**Resolution limit, sketched.** Because the null term scales with the *whole* network's $2m$, two small but
clearly separate modules can look like noise relative to the global expectation and get merged once their
internal links drop below $\sim\sqrt{2m}$ — a global yardstick applied to local structure.

:::
