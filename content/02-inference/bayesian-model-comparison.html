Bayesian Model Checking & Comparison

  • Show All Code
  • Hide All Code
  • View Source

Fitting a model is not believing it. Two questions remain — does it fit the data you have, and which model predicts the data you don’t? — and the right tools are not the famous ones.

A posterior is an answer conditional on the model being true. Before you trust it you must ask does the model fit? (checking) and, among rivals, which predicts best? (comparison). Bayesian practice answers the first with posterior predictive checks — simulate fake data and see if it looks real — and the second with out-of-sample predictive accuracy (LOO-CV, WAIC), not with the seductive-but-treacherous Bayes factor.

Why it matters

The most expensive Bayesian mistakes come from skipping these steps: trusting a model that never fit, or selecting among models with a tool (Bayes factors) that quietly lets the prior drive the verdict. This entry gives the working checklist — what to run, what each catches, and what to distrust — closing the Bayesian sub-track with the discipline that keeps it honest.

Posterior predictive checks — your best friend

The posterior predictive distribution generates the data you might collect tomorrow if the data-generating process holds (Lambert 2018). You sample it in two steps (Lambert 2018):

  1. Draw a parameter from the posterior, \theta^{(i)} \sim p(\theta\mid\text{data}).
  2. Draw a new dataset from the likelihood, x'^{(i)} \sim p(x'\mid\theta^{(i)}).

The logic is disarmingly simple: if the model fits well, simulated data should be indistinguishable from the real data (Lambert 2018). You compare them graphically, or via test statistics — the minimum, maximum, or proportion of zeros are common choices (Lambert 2018). The Bayesian p-value is the fraction of simulated datasets whose statistic is as extreme or more extreme than the observed one; near 0.5 is healthy, near 0 or 1 signals misfit (Lambert 2018). Lambert calls PPCs your “best friend” because they are superb at “hinting at a model’s inadequacies” (Lambert 2018). (The prior predictive distribution is the same idea using prior draws — the data your model can generate before seeing any.) (Lambert 2018)

Show the figure code 
import numpy as np
import matplotlib.pyplot as plt

rng = np.random.default_rng(5)
n = 200
# observed data: zero-inflated counts (40% structural zeros, else Poisson(3))
structural_zero = rng.random(n) < 0.40
y_obs = np.where(structural_zero, 0, rng.poisson(3, n))

lam = y_obs.mean()                       # fit a plain Poisson matched to the mean
S = 2000
reps = rng.poisson(lam, size=(S, n))     # replicated datasets from the fitted model

prop_zero_rep = (reps == 0).mean(axis=1)
mean_rep = reps.mean(axis=1)
prop_zero_obs = (y_obs == 0).mean()
mean_obs = y_obs.mean()
p_zeros = np.mean(prop_zero_rep >= prop_zero_obs)
p_mean = np.mean(mean_rep >= mean_obs)

fig, (axL, axR) = plt.subplots(1, 2, figsize=(8.8, 3.8))
axL.hist(prop_zero_rep, bins=30, color="#cdd6df", edgecolor="white")
axL.axvline(prop_zero_obs, color="#c0392b", lw=2.4)
axL.set_title(f"T = proportion of zeros\nBayesian p ≈ {p_zeros:.2f}  → MISFIT", fontsize=10)
axL.set_xlabel("proportion of zeros in replicated data")
axL.text(prop_zero_obs-0.01, axL.get_ylim()[1]*0.6, "observed", color="#c0392b",
         ha="right", fontsize=8.5, rotation=90)

axR.hist(mean_rep, bins=30, color="#cdd6df", edgecolor="white")
axR.axvline(mean_obs, color="#1f8a4c", lw=2.4)
axR.set_title(f"T = mean\nBayesian p ≈ {p_mean:.2f}  → looks fine", fontsize=10)
axR.set_xlabel("mean of replicated data")
fig.suptitle("Posterior predictive check: the right statistic exposes the misfit", fontsize=11.5)
fig.tight_layout()
plt.show()

Bayes factors — and why to distrust them

A Bayes factor treats the model itself as a parameter and compares two models by the ratio of their marginal likelihoods (the “evidence”) (Lambert 2018):

\text{BF}_{1,0} = \frac{p(\text{data}\mid M_1)}{p(\text{data}\mid M_0)} .

Lambert reads it on Jeffreys’ scale: 1–3 weak, 3–10 moderate, 10–100 strong, >100 “decisive” evidence for M_1 (Lambert 2018). But the weaknesses are severe (Lambert 2018):

  • Intractability. The marginal likelihood is the same high-dimensional integral that defeats exact Bayesian inference (Lambert 2018).
  • Prior sensitivity (Jeffreys–Lindley paradox). BFs are extremely sensitive to the prior even when the posterior is not — and a very diffuse prior will always unfairly favour the simpler model (Lambert 2018).
  • Undefined under improper priors. The arbitrary normalising constant does not cancel, so a researcher can change the answer at will (Lambert 2018).

The practical upshot: prefer predictive criteria below for routine model comparison; reserve Bayes factors for cases with genuinely informative, proper priors.

Information criteria: estimating out-of-sample fit

Information criteria estimate out-of-sample predictive performance from in-sample data, correcting the bias of using the data twice (Lambert 2018):

  • Deviance = -2\ln L — the raw badness-of-fit (Lambert 2018).
  • DIC (Deviance Information Criterion) — a Bayesian AIC using the posterior-mean estimate, penalised by the effective number of parameters p_D = 2\operatorname{Var}_{\text{posterior}}(\ln p(y\mid\theta)) (Lambert 2018).
  • WAIC (Widely Applicable Information Criterion) — the fully Bayesian, pointwise criterion: it treats each data point separately and uses the full posterior (not a point estimate), with a penalty equal to the sum of the posterior variances of the pointwise log-likelihood (Lambert 2018). WAIC is the modern default.

Cross-validation: LOO and PSIS-LOO

Cross-validation is the “ideal measure” of predictive accuracy — it uses a genuinely held-out point (Lambert 2018). Leave-one-out (LOO-CV) trains on n-1 points and tests on the one left out, repeated for all n (Lambert 2018). The target is the elpd — the expected log pointwise predictive density, the distance between the model’s predictions and the true data-generating distribution (Lambert 2018). Refitting n times is expensive, so Lambert recommends PSIS-LOO (Pareto-Smoothed Importance Sampling), which approximates LOO from a single posterior fit using importance weights (Lambert 2018). (Its Pareto-k diagnostic flags points where the approximation is unreliable.)

Occam’s razor and the bias–variance balance

Model comparison is a balance between fitting this sample and generalising to the next (Lambert 2018):

  • Bayesian Occam’s razor. A complex model must spread its probability mass thinly over many possible datasets, so it is “punished” for wasting probability on outcomes that don’t occur — simpler models that concentrate mass on what did happen are automatically favoured (Lambert 2018).
  • Bias–variance. Complex models can eliminate bias but inflate variance, making predictions hypersensitive to the training sample (Lambert 2018) — the bias–variance trade-off in Bayesian dress.
  • Overfitting = cheating. Lambert’s image: overfitting is “stealing the answers before a test” — it performs perfectly on old data and “fails miserably on new data” (Lambert 2018).
TipA practical hierarchy

Always run posterior predictive checks — they catch misfit no number can. For comparing models, prefer LOO-CV / WAIC (predictive accuracy). Treat Bayes factors with suspicion unless your priors are proper and meaningful. And let Occam break ties toward the simpler model. (Lambert 2018)

Common pitfalls

WarningWhere this goes wrong
  • Skipping PPCs. A converged chain on a misspecified model gives precise, confident, wrong answers — the check is non-optional. (Lambert 2018)
  • Selecting by Bayes factor under vague priors. Jeffreys–Lindley means the diffuse prior decides; you’ll systematically over-favour the null/simple model. (Lambert 2018)
  • Computing a Bayes factor with improper priors. The result is arbitrary — don’t report it. (Lambert 2018)
  • Reading WAIC/LOO differences without their standard error. Small elpd gaps are within noise; compare the difference to its SE before declaring a winner. (Lambert 2018)
  • Choosing the model with the best in-sample fit. That rewards overfitting; judge by out-of-sample prediction. (Lambert 2018)

Appendix · the mathematics

Bayesian p-value, defined. For a test statistic T, p_B = \Pr\!\big(T(y^{\text{rep}},\theta) \ge T(y,\theta)\mid y\big), averaged over the posterior of \theta and the replicated y^{\text{rep}}. Unlike a frequentist p-value it is not uniform under the null — but extreme values (≈0 or 1) still flag the statistics the model fails to reproduce.

WAIC. With pointwise log predictive density \text{lppd} = \sum_i \log \mathbb{E}_{\text{post}}[p(y_i\mid\theta)] and penalty p_{\text{WAIC}} = \sum_i \operatorname{Var}_{\text{post}}(\log p(y_i\mid\theta)), \text{WAIC} = -2(\text{lppd} - p_{\text{WAIC}}). Both terms are computed directly from posterior draws — no point estimate, no plug-in — which is what makes it “fully Bayesian.”

elpd and LOO. The leave-one-out estimate is \text{elpd}_{\text{loo}} = \sum_i \log p(y_i\mid y_{-i}), where p(y_i\mid y_{-i}) is the predictive density for point i from a fit excluding it. PSIS approximates each p(y_i\mid y_{-i}) by reweighting the full-fit posterior draws with importance weights \propto 1/p(y_i\mid\theta^{(s)}), Pareto-smoothed for stability.

The Bayesian Occam factor. The evidence factorises as (best-fit likelihood) × (Occam factor), where the Occam factor \approx posterior width / prior width < 1 — a model with many parameters or wide priors pays a larger penalty because it spread its prior predictive mass over more datasets. This is the automatic parsimony Bayes factors encode (and also why they depend so heavily on prior width).

  the difference to its SE before declaring a winner. [@lambert_bayes]
- **Choosing the model with the best *in-sample* fit.** That rewards overfitting; judge by out-of-sample
  prediction. [@lambert_bayes]
:::

## Appendix · the mathematics {.unnumbered}

::: {.math-appendix}

**Bayesian $p$-value, defined.** For a test statistic $T$,
$p_B = \Pr\!\big(T(y^{\text{rep}},\theta) \ge T(y,\theta)\mid y\big)$, averaged over the posterior of
$\theta$ and the replicated $y^{\text{rep}}$. Unlike a frequentist $p$-value it is *not* uniform under the
null — but extreme values (≈0 or 1) still flag the statistics the model fails to reproduce.

**WAIC.** With pointwise log predictive density
$\text{lppd} = \sum_i \log \mathbb{E}_{\text{post}}[p(y_i\mid\theta)]$ and penalty
$p_{\text{WAIC}} = \sum_i \operatorname{Var}_{\text{post}}(\log p(y_i\mid\theta))$,
$\text{WAIC} = -2(\text{lppd} - p_{\text{WAIC}})$. Both terms are computed directly from posterior draws —
no point estimate, no plug-in — which is what makes it "fully Bayesian."

**elpd and LOO.** The leave-one-out estimate is
$\text{elpd}_{\text{loo}} = \sum_i \log p(y_i\mid y_{-i})$, where $p(y_i\mid y_{-i})$ is the predictive
density for point $i$ from a fit *excluding* it. PSIS approximates each $p(y_i\mid y_{-i})$ by reweighting
the full-fit posterior draws with importance weights $\propto 1/p(y_i\mid\theta^{(s)})$, Pareto-smoothed for
stability.

**The Bayesian Occam factor.** The evidence factorises as (best-fit likelihood) × (Occam factor), where the
Occam factor $\approx$ posterior width / prior width $< 1$ — a model with many parameters or wide priors
pays a larger penalty because it spread its prior predictive mass over more datasets. This is the *automatic*
parsimony Bayes factors encode (and also why they depend so heavily on prior width).

:::
