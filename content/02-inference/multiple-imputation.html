Missing Data & Multiple Imputation

  • Show All Code
  • Hide All Code
  • View Source

Real datasets have holes. The two reflexes — drop the rows, or fill in the mean — quietly bias your estimates and fake your certainty. The principled fix starts by asking why the data is missing.

Every real dataset has missing values, and the software default — delete any row with a hole — can be a “grave threat” to your conclusions. Filling holes with the column mean is worse: it shrinks variance, flattens relationships, and pretends a guess is a fact. The honest approach has two parts: diagnose why the data is missing (the mechanism), then use multiple imputation — fill the holes several times, with noise, and combine — so the final uncertainty includes the fact that you were guessing.

Why it matters

Missingness is not a nuisance to be cleared before “the real analysis” — it is part of the analysis. Get it wrong and you bias estimates, lose power, and report confidence intervals that are too narrow. Get it right and you recover most of what careless deletion throws away, with honest uncertainty. This entry sits in inference because it is fundamentally about propagating the uncertainty of what you didn’t observe.

The three mechanisms

What you’re allowed to do depends entirely on why values are missing (Murphy 2012; Gerber and Green 2012):

  • MCAR — Missing Completely At Random. Missingness is independent of both observed and unobserved data — “unpredictable, no systematic pattern” (Gerber and Green 2012). The observed data is a random subsample, so estimates stay unbiased (you just lose power) (Gerber and Green 2012).
  • MAR — Missing At Random. Missingness depends only on observed data, not the missing value itself (Murphy 2012). Example: men report income less often, but conditional on gender it’s random. Under MAR you can recover unbiased estimates by conditioning on the relevant covariates (Murphy 2012).
  • MNAR — Missing Not At Random. Missingness depends on the unobserved value itself — the “worst type” because it is informative (very high or very low earners skipping the income question) (Gerber and Green 2012). Standard fixes fail; the missingness pattern is itself a rival explanation (Gerber and Green 2012).

The crux: MCAR and MAR are handle-able; MNAR needs a model of the missingness (or honest caveats). Most methods, including multiple imputation, assume MAR.

Why the naive fixes fail

Show the figure code 
import numpy as np
import matplotlib.pyplot as plt

rng = np.random.default_rng(2)
n = 200
x = rng.uniform(0, 10, n)
y_full = 1.0 + 0.8*x + rng.normal(0, 1.5, n)
# MAR: probability of missingness rises with x (depends on OBSERVED x, not y)
p_miss = 1/(1 + np.exp(-(x - 6)))
miss = rng.random(n) < p_miss
obs = ~miss

def slope(xx, yy):
    return np.polyfit(xx, yy, 1)[0]

# mean imputation
y_mean = y_full.copy(); y_mean[miss] = y_full[obs].mean()
# single MI draw: regress y~x on observed, impute = prediction + noise
b1, b0 = np.polyfit(x[obs], y_full[obs], 1)
resid_sd = np.std(y_full[obs] - (b0 + b1*x[obs]), ddof=2)
y_mi = y_full.copy()
y_mi[miss] = b0 + b1*x[miss] + rng.normal(0, resid_sd, miss.sum())

fig, (axL, axR) = plt.subplots(1, 2, figsize=(9, 4.0), sharey=True)
xs = np.linspace(0, 10, 30)
for ax in (axL, axR):
    ax.plot(xs, 1.0 + 0.8*xs, "k--", lw=1.4, alpha=0.7)
    ax.scatter(x[obs], y_full[obs], s=12, color="#9aa0a6", alpha=0.5, label="observed")
    ax.set_xlabel("x")
axL.scatter(x[miss], y_mean[miss], s=18, color="#c0392b", label="mean-imputed")
axL.set_title(f"Mean imputation  (slope {slope(x, y_mean):.2f})"); axL.set_ylabel("y")
axL.legend(fontsize=8, frameon=False, loc="upper left")
axR.scatter(x[miss], y_mi[miss], s=18, color="#1f8a4c", label="MI-imputed (1 draw)")
axR.set_title(f"Multiple imputation  (slope {slope(x, y_mi):.2f})")
axR.legend(fontsize=8, frameon=False, loc="upper left")
fig.suptitle(f"True slope = 0.80.  Mean imputation flattens it; MI preserves it.", fontsize=11)
fig.tight_layout()
plt.show()
  • Complete-case (listwise) deletion — the software default — drops any row with a single missing value (Gerber and Green 2012). It is a “grave threat” if data aren’t MCAR (the survivors are unrepresentative), and it shrinks the effective sample, inflating standard errors and sacrificing power (Gerber and Green 2012).
  • Single / mean imputation replaces a hole with the average (Murphy 2012). It artificially reduces variance, lowers correlations, and — fatally — “understates uncertainty” by treating a guess as a known value (Murphy 2012). The left panel above is this failure made visible.

Multiple imputation (Rubin)

Multiple imputation (MI) fixes single imputation’s lie by admitting we don’t know the missing values (Murphy 2012):

  1. Create M completed datasets. In each, draw the missing entries from a posterior predictive distribution p(x_{\text{hidden}}\mid x_{\text{visible}},\theta) that includes random noise to reflect uncertainty (Murphy 2012).
  2. Analyse each separately with standard methods (e.g. OLS), giving M estimates and M standard errors (Murphy 2012).
  3. Combine with Rubin’s rules (Murphy 2012):
    • Point estimate = the average of the M estimates.
    • Total variance = the within-imputation variance (average of the M variances — ordinary sampling error) plus (1+\tfrac1M)\times the between-imputation variance (the variance of the M estimates). That second term is the magic: it explicitly captures the extra uncertainty created by the missingness itself (Murphy 2012).

This is why MI’s confidence intervals are honestly wider than mean imputation’s — they remember you were guessing.

MICE: chained equations

The workhorse algorithm is MICE (Multivariate Imputation by Chained Equations) (Murphy 2012). It handles many missing variables of mixed types by specifying a regression for each variable conditional on all the others, then cycling through the variables — replacing each one’s missing values with draws from its model — until the imputations converge to a stable joint distribution (Murphy 2012).

Practical guidance

  • Include auxiliary variables. Put variables that predict missingness into the imputation model even if they won’t be in the final analysis — it makes the MAR assumption more plausible (Murphy 2012).
  • Impute, then transform. Do logs/√ after imputation (or build them into the imputation model) so you don’t distort relationships between variables (Murphy 2012).
  • Use enough imputations. The old advice of M=5 is superseded — modern computing makes M=20+ cheap and gives stable between-imputation variance (Murphy 2012).
  • Report a sensitivity analysis. Because MAR can never be proven, it’s good practice to report both the complete-case and imputed results and check they agree (Gerber and Green 2012).

Common pitfalls

WarningWhere this goes wrong
  • Deleting rows by default. Complete-case analysis is biased unless MCAR and always wastes power; justify it or impute. (Gerber and Green 2012)
  • Mean (single) imputation. It fakes certainty and attenuates relationships — never use it for inference. (Murphy 2012)
  • Imputing the outcome and then “finding” effects. Including the analysis variables but omitting key predictors of missingness undercuts the MAR assumption. (Murphy 2012)
  • Assuming MAR when MNAR is plausible. If the value itself drives its missingness (income, symptom severity), standard MI is biased — model the missingness or caveat heavily. (Gerber and Green 2012)
  • Reporting MI point estimates with naive SEs. You must use Rubin’s rules; the between-imputation variance is the whole point. (Murphy 2012)

Appendix · the mathematics

Rubin’s rules, formally. With per-imputation estimates \hat\theta_m and variances U_m (m=1,\dots,M):

\bar\theta = \frac{1}{M}\sum_m \hat\theta_m,\qquad \bar U = \frac{1}{M}\sum_m U_m \ \text{(within)},\qquad B = \frac{1}{M-1}\sum_m (\hat\theta_m-\bar\theta)^2 \ \text{(between)}.

The total variance is T = \bar U + \big(1+\tfrac1M\big)B. As B\to0 (the imputations agree, little missing-data uncertainty) T\to\bar U, the ordinary variance; as B grows, the intervals widen. The (1+1/M) factor corrects for using a finite number of imputations. Inference uses a t-distribution with Rubin’s degrees of freedom \nu = (M-1)\big(1+\bar U/((1+1/M)B)\big)^2.

Why mean imputation shrinks variance. Replacing n_{\text{mis}} values with the constant \bar x adds points with zero deviation from the mean, so the sample variance \frac{1}{n-1}\sum(x_i-\bar x)^2 is computed with extra zero terms in the numerator but a larger denominator — it is biased downward, and any correlation with another variable is attenuated toward zero. MI avoids this by imputing draws with the right residual spread.

$$

The **total variance** is $T = \bar U + \big(1+\tfrac1M\big)B$. As $B\to0$ (the imputations agree, little
missing-data uncertainty) $T\to\bar U$, the ordinary variance; as $B$ grows, the intervals widen. The
$(1+1/M)$ factor corrects for using a **finite** number of imputations. Inference uses a $t$-distribution
with Rubin's degrees of freedom $\nu = (M-1)\big(1+\bar U/((1+1/M)B)\big)^2$.

**Why mean imputation shrinks variance.** Replacing $n_{\text{mis}}$ values with the constant $\bar x$ adds
points with **zero deviation** from the mean, so the sample variance
$\frac{1}{n-1}\sum(x_i-\bar x)^2$ is computed with extra zero terms in the numerator but a larger
denominator — it is **biased downward**, and any correlation with another variable is attenuated toward zero.
MI avoids this by imputing **draws with the right residual spread**.
