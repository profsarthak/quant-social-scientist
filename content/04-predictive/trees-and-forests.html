Trees, Forests & Boosting

  • Show All Code
  • Hide All Code
  • View Source

A flowchart anyone can read, its fatal flakiness, and the ensembles that turn a weak learner into a champion.

A decision tree asks a series of yes/no questions and reads off an answer — the most interpretable model there is. It is also dangerously high-variance: redraw the data and the tree reshuffles. The fix is to grow hundreds of trees and average them. That move — random forests and boosting — sacrifices the flowchart for some of the best prediction accuracy available.

Why it matters

Trees are the other great route (besides regularization) to managing the bias–variance trade-off, and they handle something linear models struggle with: non-linearities and interactions you never specified. (James et al. 2021) A single tree is a gift for communication — you can show it to a policymaker. Its ensembles are a gift for accuracy — forests and boosting are among the strongest off-the-shelf predictors on tabular social data. (Bruce et al. 2020) The journey from one tree to a forest is also the cleanest illustration of why averaging decorrelated models is the deepest trick in predictive modelling.

Decision trees: a flowchart fit to data

A tree predicts by recursive binary splitting — a greedy, top-down carve-up of the predictor space. (James et al. 2021) Start at the root (all observations); find the single predictor and cutpoint whose split most improves fit; repeat inside each branch; stop at terminal nodes (leaves), where the prediction is just the mean (regression) or majority class (classification) of the observations there. (James et al. 2021) How is “best split” judged? Regression trees minimise RSS; classification trees maximise node purity via the Gini index or cross-entropy (better than raw error rate for growing the tree). (James et al. 2021)

NoteISL’s baseball-salary tree

Predicting log-salary from a player’s career stats, a small 3-leaf tree uses just Years and Hits: players with < 4.5 years experience average about $165,174; experienced players with ≥ 118 hits average $845,346. (James et al. 2021) Two questions, a clear story — “pay rises with experience, then with productivity” — readable by anyone. Trees are even easier to explain than linear regression, and arguably mirror how humans actually decide. (James et al. 2021)

The flaw: high variance, and pruning

A single tree’s fatal weakness: it is high-variance and overfits easily. (James et al. 2021) Grow it deep enough and it carves the data into perfectly pure leaves — memorising the noise — and a slightly different sample yields a wholly different tree. Pruning (cost-complexity / weakest-link pruning) controls this: grow a large tree, then cut it back with a penalty \alpha on the number of leaves, choosing \alpha by cross-validation. (James et al. 2021) But pruning only mitigates the variance — the real cure is to stop relying on one tree.

See it: one jagged tree vs a smooth forest

Below, a single deep tree (orange) fits the data as a wild step function — over-reacting to every wiggle of noise. A forest of many trees, each grown on a bootstrap resample and then averaged (blue), smooths those steps into a stable estimate of the signal. Same data, the variance averaged away.

Show the figure code (a minimal regression tree + bagging) 
import numpy as np
import matplotlib.pyplot as plt

rng = np.random.default_rng(0)

def fit_tree(x, y, depth, max_depth, min_leaf=3):
    if depth >= max_depth or x.size < 2 * min_leaf:
        m = y.mean();  return lambda xq: np.full(xq.shape, m)
    order = np.argsort(x); xs = x[order]
    best = (np.inf, None)
    for i in range(min_leaf, x.size - min_leaf + 1):
        t = 0.5 * (xs[i-1] + xs[i])
        l, r = y[x <= t], y[x > t]
        if l.size and r.size:
            sse = ((l - l.mean())**2).sum() + ((r - r.mean())**2).sum()
            if sse < best[0]: best = (sse, t)
    t = best[1]
    if t is None:
        m = y.mean();  return lambda xq: np.full(xq.shape, m)
    L = fit_tree(x[x <= t], y[x <= t], depth+1, max_depth)
    R = fit_tree(x[x > t],  y[x > t],  depth+1, max_depth)
    return lambda xq: np.where(xq <= t, L(xq), R(xq))

n = 70
x = np.sort(rng.uniform(0, 10, n))
signal = lambda x: np.sin(0.9 * x) + 0.25 * x
y = signal(x) + rng.normal(0, 0.45, n)
grid = np.linspace(0, 10, 400)

single = fit_tree(x, y, 0, max_depth=10)(grid)               # one deep tree
B = 150
forest = np.zeros_like(grid)
for _ in range(B):
    idx = rng.integers(0, n, n)                               # bootstrap resample
    forest += fit_tree(x[idx], y[idx], 0, max_depth=10)(grid)
forest /= B

fig, ax = plt.subplots(figsize=(7, 4.2))
ax.scatter(x, y, s=16, color="#9aa0a6", alpha=0.6, zorder=2)
ax.plot(grid, signal(grid), color="#7a7f87", lw=1.6, ls="--", label="true signal")
ax.plot(grid, single, color="#d98a29", lw=1.6, label="single deep tree (overfit)")
ax.plot(grid, forest, color="#1f3a5f", lw=2.6, label=f"forest: average of {B} trees")
ax.set_xlabel("x"); ax.set_ylabel("y")
ax.set_title("One tree is jagged; a forest is smooth")
ax.legend(fontsize=8, frameon=False, loc="upper left")
fig.tight_layout()
plt.show()

Bagging: average away the variance

Bagging (bootstrap aggregating) is the general cure for any high-variance method. (James et al. 2021) Draw B bootstrap resamples, grow a deep (unpruned) tree on each, and average their predictions (or take a majority vote). Averaging many noisy-but-unbiased estimates slashes variance without adding bias — the law of large numbers applied to models. A free bonus: out-of-bag (OOB) error. Each tree omits about a third of the data (the observations not drawn into its bootstrap), so predicting each point with only the trees that didn’t see it gives a built-in test-error estimate — no separate validation set needed. (James et al. 2021)

Random forests: decorrelate, then average

Bagging has a subtle weakness, and random forests fix it. (James et al. 2021) If one predictor is very strong, every bagged tree splits on it first — so the trees are highly correlated, and averaging correlated things barely reduces variance (the same correlation penalty as in LOOCV). The fix is elegantly simple: at each split, allow only a random subset of m predictors to be considered. (James et al. 2021) This forces trees to use different variables, decorrelating them, so their average is far more stable. The standard choice is m \approx \sqrt{p}. (James et al. 2021) That one tweak — randomly handicapping each split — is what makes a random forest beat plain bagging.

Boosting: learn slowly from your mistakes

Boosting takes the opposite philosophy to bagging’s parallel averaging: it grows trees sequentially, each new tree fitting the residuals (the errors) of the current ensemble. (James et al. 2021) The model learns slowly, concentrating on the cases it has been getting wrong. Three tuning knobs: the number of trees B (too many can overfit, but slowly), the shrinkage / learning rate \lambda (how fast it learns — typically 0.01 or 0.001), and the tree depth d (often tiny — even stumps, d=1, suffice). (James et al. 2021) Well-tuned boosting is frequently the most accurate method on structured/tabular data.

Peeking into the black box: variable importance

Ensembles buy accuracy by sacrificing the single tree’s interpretability — there’s no one flowchart to read. (James et al. 2021) Variable importance measures partly reopen the box: (James et al. 2021)

  • Gini / RSS importance — total up how much each predictor’s splits reduced RSS (regression) or the Gini index (classification), averaged over all trees.
  • Permutation importance — randomly shuffle a predictor’s values and measure how much accuracy drops; a big drop means the model relied on it.
NoteISL’s Heart data

For predicting heart disease, the most important predictors surfaced by a forest were Thal, Ca, and ChestPain. (James et al. 2021) Importance gives a ranking — useful and suggestive — but, as the next entry stresses, a ranking is not a causal effect. (James et al. 2021)

Trees vs linear models: which wins?

It depends entirely on the true data-generating process. (James et al. 2021)

  • Linear/logistic models win when the relationship is approximately linear — OLS exploits that structure efficiently, and trees only approximate a line with clumsy steps.
  • Trees and forests win when the truth is highly non-linear with many interactions the researcher never specified — they discover that structure automatically. (James et al. 2021)

A useful division of labour: single trees for communication and exploration; forests and boosting for maximum predictive power. (Bruce et al. 2020)

Common pitfalls

WarningWhere this goes wrong

  • Trusting a single deep tree. It’s high-variance and overfit; a different sample gives a different tree. Prune it, or move to an ensemble. (James et al. 2021)

  • Expecting a forest to be interpretable. Averaging hundreds of trees destroys the flowchart. Use variable importance / partial dependence, and accept you’ve traded transparency for accuracy. (James et al. 2021)

  • Reading variable importance as a causal effect. It measures predictive contribution, not cause; a strong predictor may be a proxy or a collider. (James et al. 2021)

  • Skipping tuning on boosting. Too high a learning rate or too-deep trees overfit; boosting needs CV on its knobs. (James et al. 2021)

  • Using trees where a line would do. On near-linear data, regression beats trees and stays interpretable. Match the method to the structure. (James et al. 2021)

Appendix · the mathematics

Split criteria. A regression split sends the data into regions R_1,R_2 to minimise \sum_{i\in R_1}(y_i-\bar y_{R_1})^2 + \sum_{i\in R_2}(y_i-\bar y_{R_2})^2. A classification split minimises a purity measure — the Gini index G=\sum_k \hat p_k(1-\hat p_k) or cross-entropy D=-\sum_k \hat p_k\log\hat p_k — both small when one class dominates a node.

Cost-complexity pruning. For a large tree T_0, choose the subtree T\subseteq T_0 minimising \sum_{m=1}^{|T|}\sum_{i\in R_m}(y_i-\bar y_{R_m})^2 + \alpha|T|, where |T| is the number of leaves and \alpha\ge 0 trades fit against size. Increasing \alpha yields a nested sequence of subtrees; CV picks \alpha.

Why decorrelation reduces variance. The average of B identically-distributed trees each with variance \sigma^2 and pairwise correlation \rho has variance \rho\,\sigma^2 + \frac{1-\rho}{B}\,\sigma^2 . As B\to\infty the second term vanishes, but the first, \rho\sigma^2, does not — so bagging is capped by how correlated the trees are. Random forests shrink \rho (by restricting splits to m random predictors), pushing the floor down — the exact mechanism behind their gain. (Same correlated-average algebra as the Moulton/clustering and LOOCV results.)

## Bagging: average away the variance

**Bagging** (bootstrap aggregating) is the general cure for any high-variance method. [@islr] Draw
$B$ [bootstrap](../02-inference/confidence-intervals.qmd) resamples, grow a deep (unpruned) tree on
each, and **average** their predictions (or take a majority vote). Averaging many noisy-but-unbiased
estimates slashes variance without adding bias — the [law of large numbers](../01-foundations/law-of-large-numbers.qmd)
applied to models. A free bonus: **out-of-bag (OOB) error**. Each tree omits about a third of the
data (the observations not drawn into its bootstrap), so predicting each point with only the trees
that *didn't* see it gives a built-in test-error estimate — no separate
[validation set](cross-validation.qmd) needed. [@islr]

## Random forests: decorrelate, then average

Bagging has a subtle weakness, and **random forests** fix it. [@islr] If one predictor is very
strong, *every* bagged tree splits on it first — so the trees are **highly correlated**, and
averaging correlated things barely reduces variance (the same [correlation penalty](cross-validation.qmd)
as in LOOCV). The fix is elegantly simple: **at each split, allow only a random subset of $m$
predictors** to be considered. [@islr] This forces trees to use *different* variables, **decorrelating**
them, so their average is far more stable. The standard choice is $m \approx \sqrt{p}$. [@islr] That
one tweak — randomly handicapping each split — is what makes a random forest beat plain bagging.

## Boosting: learn slowly from your mistakes

Boosting takes the opposite philosophy to bagging's parallel averaging: it grows trees
**sequentially**, each new tree fitting the **residuals** (the errors) of the current ensemble.
[@islr] The model **learns slowly**, concentrating on the cases it has been getting wrong. Three
tuning knobs: the **number of trees** $B$ (too many *can* overfit, but slowly), the **shrinkage /
learning rate** $\lambda$ (how fast it learns — typically 0.01 or 0.001), and the **tree depth** $d$
(often tiny — even **stumps**, $d=1$, suffice). [@islr] Well-tuned boosting is frequently the most
accurate method on structured/tabular data.

## Peeking into the black box: variable importance

Ensembles buy accuracy by **sacrificing the single tree's interpretability** — there's no one
flowchart to read. [@islr] **Variable importance** measures partly reopen the box: [@islr]

- **Gini / RSS importance** — total up how much each predictor's splits reduced RSS (regression) or
  the Gini index (classification), averaged over all trees.
- **Permutation importance** — randomly **shuffle** a predictor's values and measure how much
  accuracy drops; a big drop means the model relied on it.

::: {.callout-note title="ISL's Heart data"}
For predicting heart disease, the most important predictors surfaced by a forest were **Thal**,
**Ca**, and **ChestPain**. [@islr] Importance gives a *ranking* — useful and suggestive — but, as the
next entry stresses, **a ranking is not a causal effect.** [@islr]
:::

## Trees vs linear models: which wins?

It depends entirely on the true data-generating process. [@islr]

- **Linear/logistic models win** when the relationship is approximately **linear** — OLS exploits that
  structure efficiently, and trees only approximate a line with clumsy steps.
- **Trees and forests win** when the truth is **highly non-linear with many interactions** the
  researcher never specified — they discover that structure automatically. [@islr]

A useful division of labour: **single trees for communication and exploration; forests and boosting
for maximum predictive power.** [@bruce_practical]

## Common pitfalls

::: {.callout-warning title="Where this goes wrong"}

- **Trusting a single deep tree.** It's high-variance and overfit; a different sample gives a
  different tree. Prune it, or move to an ensemble. [@islr]

- **Expecting a forest to be interpretable.** Averaging hundreds of trees destroys the flowchart.
  Use variable importance / [partial dependence](interpretability-and-ml.qmd), and accept you've
  traded transparency for accuracy. [@islr]

- **Reading variable importance as a causal effect.** It measures predictive contribution, not
  cause; a strong predictor may be a proxy or a collider. [@islr]

- **Skipping tuning on boosting.** Too high a learning rate or too-deep trees overfit; boosting needs
  CV on its knobs. [@islr]

- **Using trees where a line would do.** On near-linear data, regression beats trees and stays
  interpretable. Match the method to the structure. [@islr]

:::

## Appendix · the mathematics {.unnumbered}

::: {.math-appendix}

**Split criteria.** A regression split sends the data into regions $R_1,R_2$ to minimise
$\sum_{i\in R_1}(y_i-\bar y_{R_1})^2 + \sum_{i\in R_2}(y_i-\bar y_{R_2})^2$. A classification split
minimises a purity measure — the **Gini index** $G=\sum_k \hat p_k(1-\hat p_k)$ or **cross-entropy**
$D=-\sum_k \hat p_k\log\hat p_k$ — both small when one class dominates a node.

**Cost-complexity pruning.** For a large tree $T_0$, choose the subtree $T\subseteq T_0$ minimising
$\sum_{m=1}^{|T|}\sum_{i\in R_m}(y_i-\bar y_{R_m})^2 + \alpha|T|$, where $|T|$ is the number of leaves
and $\alpha\ge 0$ trades fit against size. Increasing $\alpha$ yields a nested sequence of subtrees;
CV picks $\alpha$.

**Why decorrelation reduces variance.** The average of $B$ identically-distributed trees each with
variance $\sigma^2$ and pairwise correlation $\rho$ has variance
$$
\rho\,\sigma^2 + \frac{1-\rho}{B}\,\sigma^2 .
$$
As $B\to\infty$ the second term vanishes, but the first, $\rho\sigma^2$, **does not** — so bagging is
capped by how correlated the trees are. Random forests shrink $\rho$ (by restricting splits to $m$
random predictors), pushing the floor down — the exact mechanism behind their gain. (Same
correlated-average algebra as the [Moulton/clustering](../03-regression/regression-diagnostics.qmd)
and [LOOCV](cross-validation.qmd) results.)

:::
